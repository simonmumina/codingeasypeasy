# Bootstrap 5 Button Groups: A Comprehensive Guide with Code Examples

Button groups are a fundamental UI component in modern web development, allowing you to group related buttons together for a clean and organized user interface. Bootstrap 5 provides a powerful and flexible way to create these button groups, offering various styling options and functionalities. This guide will walk you through everything you need to know about Bootstrap 5 button groups, complete with practical code examples.

## What are Button Groups?

A button group is essentially a container holding a set of buttons. These buttons are typically related in function, allowing users to perform a series of actions within a specific context. For example, a button group might contain buttons for "Edit," "Save," and "Cancel" operations.

## Why Use Bootstrap 5 Button Groups?

*   **Clean and Organized UI:**  Groups related actions together, improving usability and visual appeal.
*   **Responsive Design:**  Button groups adapt to different screen sizes, ensuring a consistent experience across devices.
*   **Customizable:** Bootstrap offers a wide range of styling options to tailor button groups to your specific design requirements.
*   **Accessibility:** Bootstrap provides built-in accessibility features for button groups, ensuring they are usable by everyone.
*   **Easy Implementation:** With Bootstrap's class-based system, creating button groups is straightforward and efficient.
*   **Consistent Design Language:** Adheres to the Bootstrap aesthetic, contributing to a cohesive look and feel for your website.

## Basic Implementation: Creating a Simple Button Group

The foundation of a Bootstrap 5 button group is the `.btn-group` class.  Wrap your buttons within a `<div>` element with this class to create a basic button group.  Each individual button uses the standard Bootstrap button classes (e.g., `.btn`, `.btn-primary`, `.btn-secondary`).

```jsx live
<div className="btn-group" role="group" aria-label="Basic example">
  <button type="button" className="btn btn-primary">Left</button>
  <button type="button" className="btn btn-primary">Middle</button>
  <button type="button" className="btn btn-primary">Right</button>
</div>
```

**Explanation:**

*   **`.btn-group`:**  This class defines the container as a button group.
*   **`role="group"`:**  This attribute provides semantic meaning to screen readers, indicating that the buttons are grouped together.
*   **`aria-label="Basic example"`:**  This attribute provides a descriptive label for screen readers, further enhancing accessibility.  Replace "Basic example" with a more descriptive label relevant to the button group's function.
*   **`<button type="button" ...>`:**  Each button uses the standard `<button>` element.  `type="button"` is important to prevent form submission in certain contexts.
*   **`.btn`:**  This class applies the basic Bootstrap button styling.
*   **`.btn-primary`:**  This class applies the primary color scheme to the button.  You can use other color variants like `.btn-secondary`, `.btn-success`, `.btn-danger`, `.btn-warning`, `.btn-info`, `.btn-light`, and `.btn-dark`.

## Button Group Orientation: Vertical Button Groups

To create a vertical button group, add the `.btn-group-vertical` class to the button group container.

```jsx live
<div className="btn-group-vertical" role="group" aria-label="Vertical button group">
  <button type="button" className="btn btn-primary">Top</button>
  <button type="button" className="btn btn-primary">Middle</button>
  <button type="button" className="btn btn-primary">Bottom</button>
</div>
```

## Button Group Sizing: Adjusting Button Group Size

Bootstrap 5 offers utility classes to control the size of button groups: `.btn-group-lg`, `.btn-group-sm`, and the default size (which is implied without any size class).  These classes are applied to the `.btn-group` or `.btn-group-vertical` container.

```jsx live
<div>
  <div className="btn-group btn-group-lg" role="group" aria-label="Large button group">
    <button type="button" className="btn btn-primary">Large</button>
    <button type="button" className="btn btn-primary">Large</button>
  </div>
  <br />
  <div className="btn-group" role="group" aria-label="Default button group">
    <button type="button" className="btn btn-primary">Default</button>
    <button type="button" className="btn btn-primary">Default</button>
  </div>
  <br />
  <div className="btn-group btn-group-sm" role="group" aria-label="Small button group">
    <button type="button" className="btn btn-primary">Small</button>
    <button type="button" className="btn btn-primary">Small</button>
  </div>
</div>
```

## Button Toolbars: Combining Multiple Button Groups

You can combine multiple button groups into a button toolbar using the `.btn-toolbar` class. This allows you to create more complex UI structures with logical groupings of actions.  Use `aria-label` on the `btn-toolbar` to provide context for screen reader users.

```jsx live
<div className="btn-toolbar" role="toolbar" aria-label="Toolbar with button groups">
  <div className="btn-group mr-2" role="group" aria-label="First group">
    <button type="button" className="btn btn-secondary">1</button>
    <button type="button" className="btn btn-secondary">2</button>
    <button type="button" className="btn btn-secondary">3</button>
    <button type="button" className="btn btn-secondary">4</button>
  </div>
  <div className="btn-group mr-2" role="group" aria-label="Second group">
    <button type="button" className="btn btn-secondary">5</button>
    <button type="button" className="btn btn-secondary">6</button>
    <button type="button" className="btn btn-secondary">7</button>
  </div>
  <div className="btn-group" role="group" aria-label="Third group">
    <button type="button" className="btn btn-secondary">8</button>
  </div>
</div>
```

**Explanation:**

*   **`.btn-toolbar`:**  This class defines the container as a button toolbar.
*   **`mr-2` (margin-right):**  This utility class adds a right margin to the button groups for spacing.  Use other margin utilities as needed.
*   Each `btn-group` within the toolbar functions as described earlier.

## Mixing Button Groups with Input Groups

Bootstrap allows seamless integration of button groups with input groups to create more interactive and functional components.  This is commonly used for search forms or custom input controls.

```jsx live
<div className="input-group">
  <input type="text" className="form-control" placeholder="Search..." aria-label="Search" />
  <div className="input-group-append">
    <button className="btn btn-outline-secondary" type="button">Go!</button>
  </div>
</div>
<br/>
<div className="input-group">
  <div className="input-group-prepend">
    <button className="btn btn-outline-secondary dropdown-toggle" type="button" data-bs-toggle="dropdown" aria-expanded="false">Options</button>
    <ul className="dropdown-menu">
      <li><a className="dropdown-item" href="#">Action</a></li>
      <li><a className="dropdown-item" href="#">Another action</a></li>
      <li><a className="dropdown-item" href="#">Something else here</a></li>
      <li><hr className="dropdown-divider" /></li>
      <li><a className="dropdown-item" href="#">Separated link</a></li>
    </ul>
  </div>
  <input type="text" className="form-control" aria-label="Text input with dropdown button" />
</div>
```

**Explanation:**

*   **`.input-group`:** The Bootstrap input group container.
*   **`.form-control`:** The input field within the group.
*   **`.input-group-append` and `.input-group-prepend`:** These classes wrap the button(s) or dropdown on the right or left side of the input field, respectively.
*   **`.btn-outline-secondary`:**  A variant of the button class that uses an outline style.
*   The second example showcases using a dropdown menu within the input group, providing even more options. Note the use of `data-bs-toggle="dropdown"` which is required for the dropdown to function.

## Button Group with Radio Buttons or Checkboxes

You can create button groups that behave like radio buttons or checkboxes, allowing users to select one or multiple options, respectively.  This requires using JavaScript to manage the state of the buttons.  However, you can create the visual appearance with Bootstrap classes.

```jsx live
<div className="btn-group" role="group" aria-label="Basic checkbox toggle button group">
  <input type="checkbox" className="btn-check" id="btncheck1" autoComplete="off" />
  <label className="btn btn-outline-primary" htmlFor="btncheck1">Checkbox 1</label>

  <input type="checkbox" className="btn-check" id="btncheck2" autoComplete="off" />
  <label className="btn btn-outline-primary" htmlFor="btncheck2">Checkbox 2</label>

  <input type="checkbox" className="btn-check" id="btncheck3" autoComplete="off" />
  <label className="btn btn-outline-primary" htmlFor="btncheck3">Checkbox 3</label>
</div>
<br/>
<div className="btn-group" role="group" aria-label="Basic radio toggle button group">
  <input type="radio" className="btn-check" name="btnradio" id="btnradio1" autoComplete="off" defaultChecked />
  <label className="btn btn-outline-primary" htmlFor="btnradio1">Radio 1</label>

  <input type="radio" className="btn-check" name="btnradio" id="btnradio2" autoComplete="off" />
  <label className="btn btn-outline-primary" htmlFor="btnradio2">Radio 2</label>

  <input type="radio" className="btn-check" name="btnradio" id="btnradio3" autoComplete="off" />
  <label className="btn btn-outline-primary" htmlFor="btnradio3">Radio 3</label>
</div>
```

**Explanation:**

*   **`.btn-check`:** This class hides the default checkbox/radio input appearance.
*   **`.btn` and `.btn-outline-primary`:**  Style the labels as buttons.  The outline variant helps visually indicate the selected state.
*   **`htmlFor`:**  This attribute links the label to the corresponding input element, allowing the label to trigger the checkbox/radio button.
*   **`autoComplete="off"`:**  Optional, but recommended to prevent browser autocomplete behavior on the hidden inputs.
*   **JavaScript is required:** This example only provides the visual structure.  You'll need to add JavaScript code to handle the selected state of the checkboxes/radio buttons. You can do this by adding an event listener to the label element and toggling a class (e.g., `active` or the appropriate Bootstrap color class) on the label and setting the `checked` property of the input field.

## Nested Button Groups: Creating Complex Structures

Bootstrap allows you to nest button groups within other button groups to create more complex and hierarchical UI elements.

```jsx live
<div className="btn-group" role="group" aria-label="Nested button group">
  <button type="button" className="btn btn-primary">1</button>
  <button type="button" className="btn btn-primary">2</button>

  <div className="btn-group" role="group">
    <button id="btnGroupDrop1" type="button" className="btn btn-primary dropdown-toggle" data-bs-toggle="dropdown" aria-expanded="false">
      Dropdown
    </button>
    <ul className="dropdown-menu" aria-labelledby="btnGroupDrop1">
      <li><a className="dropdown-item" href="#">Dropdown link</a></li>
      <li><a className="dropdown-item" href="#">Dropdown link</a></li>
    </ul>
  </div>
</div>
```

**Explanation:**

*   This example shows a standard button group with two regular buttons and a third button that functions as a dropdown toggle.
*   The dropdown menu is nested within the outer button group.

## Accessibility Considerations

*   **`role="group"` and `aria-label`:** As demonstrated in the examples, always use these attributes to provide semantic meaning and context for screen reader users.  Ensure the `aria-label` is descriptive and accurately reflects the purpose of the button group.
*   **Keyboard Navigation:** Ensure that users can navigate through the buttons within the group using the keyboard (typically with the Tab key).  Focus styling should be clear and consistent.
*   **Color Contrast:** Maintain sufficient color contrast between the button text and background to ensure readability for users with visual impairments.
*   **ARIA Attributes for State:** When using JavaScript to manage the state of button groups (e.g., for radio buttons or checkboxes), use ARIA attributes like `aria-checked` to indicate the selected state to assistive technologies.

## Customization and Styling

Bootstrap 5 button groups are highly customizable. You can adjust the appearance of the buttons within the group using standard Bootstrap utility classes, custom CSS, or by overriding Bootstrap's default styles.  Consider these customization options:

*   **Color Variations:**  Use different `.btn-*` classes to change the color scheme of the buttons.
*   **Outline Buttons:**  Use `.btn-outline-*` classes for buttons with an outline style.
*   **Custom CSS:**  Add your own CSS rules to override Bootstrap's default styles and create a unique look and feel for your button groups.
*   **Font Styling:** Customize the font size, font weight, and font family of the button text.
*   **Border Radius:** Adjust the border radius of the buttons to create rounded or square corners.
*   **Box Shadows:** Add box shadows to the buttons for a more three-dimensional effect.
*   **Transitions:**  Use CSS transitions to create smooth animations when the buttons are hovered over or clicked.

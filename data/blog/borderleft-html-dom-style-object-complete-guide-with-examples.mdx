# BorderLeft HTML DOM Style Object: Complete Guide with Examples

The `borderLeft` property in the HTML DOM Style object is a powerful tool for dynamically controlling the left border of HTML elements using JavaScript. It allows you to read, modify, and manipulate the `border-left` CSS property, giving you granular control over the appearance of your web pages. This guide provides a comprehensive overview of the `borderLeft` property, its syntax, and practical code examples to help you master its usage.

## What is the `borderLeft` Property?

The `borderLeft` property represents the `border-left` CSS property, which is a shorthand property for setting the width, style, and color of an element's left border in a single declaration. This is different from the more granular properties `borderLeftWidth`, `borderLeftStyle`, and `borderLeftColor`. The `borderLeft` property allows you to set all three with a single statement.

## Syntax

To access and manipulate the `borderLeft` property, you use the following syntax:

```javascript
element.style.borderLeft = 'width style color'
```

Where:

- `element` is a reference to the HTML element you want to style.
- `style` is the `style` property of the element, which contains the inline styles.
- `borderLeft` is the property you are setting.
- `"width style color"` is a string representing the desired border properties. The order is important.

You can also read the current value of the `borderLeft` property:

```javascript
let borderLeftValue = element.style.borderLeft
```

## Understanding the `borderLeft` Value

The `borderLeft` value is a string that typically consists of three components:

1.  **`width`:** Specifies the width of the border. It can be a length value (e.g., "1px", "5em", "0.2cm") or one of the predefined values: `thin`, `medium`, or `thick`.

2.  **`style`:** Specifies the style of the border. Common values include:

    - `none`: No border.
    - `hidden`: Same as `none`, except in table border conflict resolution.
    - `dotted`: Dotted border.
    - `dashed`: Dashed border.
    - `solid`: Solid border.
    - `double`: Double border.
    - `groove`: 3D grooved border.
    - `ridge`: 3D ridged border.
    - `inset`: 3D inset border.
    - `outset`: 3D outset border.

3.  **`color`:** Specifies the color of the border. It can be a color name (e.g., "red", "blue", "green"), a hexadecimal color code (e.g., "#FF0000", "#00FF00", "#0000FF"), an RGB value (e.g., "rgb(255, 0, 0)", "rgb(0, 255, 0)", "rgb(0, 0, 255)"), an RGBA value (e.g., "rgba(255, 0, 0, 0.5)", "rgba(0, 255, 0, 0.5)", "rgba(0, 0, 255, 0.5)"), an HSL value (e.g., "hsl(0, 100%, 50%)"), or an HSLA value (e.g., "hsla(0, 100%, 50%, 0.5)").

**Important Considerations:**

- The order of the values (width, style, color) **matters**.
- If you omit any of these values, the browser will use its default values for the missing components. For example, if you only specify the width, the style will default to `none` and the color will be the element's text color.
- Setting `borderLeft` to an empty string (`""`) will effectively remove the inline `border-left` style, allowing the element to inherit the border from a CSS stylesheet or its parent element.

## Code Examples

Here are some practical examples demonstrating how to use the `borderLeft` property in JavaScript:

**1. Setting the border-left style:**

```plaintext
<!DOCTYPE html>
<html>
  <head>
    <title>borderLeft Example</title>
  </head>
  <body>
    <div id="myDiv" style="width: 200px; height: 100px; background-color: lightblue;">
      This is a div element.
    </div>

    <button onclick="setBorder()">Set Border Left</button>

    <script>
      function setBorder() {
        const element = document.getElementById('myDiv')
        element.style.borderLeft = '5px solid red'
      }
    </script>
  </body>
</html>
```

In this example, clicking the button sets a 5px solid red border on the left side of the `div` element.

**2. Reading the border-left style:**

```plaintext
<!DOCTYPE html>
<html>
  <head>
    <title>Reading borderLeft Value</title>
    <style>
      #myDiv {
        width: 200px;
        height: 100px;
        background-color: lightblue;
        border-left: 2px dashed blue; /* Initial border style */
      }
    </style>
  </head>
  <body>
    <div id="myDiv">This is a div element.</div>

    <button onclick="getBorder()">Get Border Left</button>
    <p id="borderValue"></p>

    <script>
      function getBorder() {
        const element = document.getElementById('myDiv')
        const borderLeftValue = element.style.borderLeft
        document.getElementById('borderValue').textContent = 'Border Left: ' + borderLeftValue
      }
    </script>
  </body>
</html>
```

This example retrieves the current `borderLeft` value of the `div` element and displays it in a paragraph. Note that the initial border is defined via an internal style.

**3. Dynamically changing the border-left style:**

```plaintext
<!DOCTYPE html>
<html>
  <head>
    <title>Dynamically Change borderLeft</title>
  </head>
  <body>
    <div id="myDiv" style="width: 200px; height: 100px; background-color: lightblue;">
      This is a div element.
    </div>

    <button onclick="changeBorder()">Change Border</button>

    <script>
      function changeBorder() {
        const element = document.getElementById('myDiv')
        const borderWidth = Math.floor(Math.random() * 10) + 1 // Random width between 1 and 10
        const borderStyles = ['solid', 'dashed', 'dotted', 'double']
        const borderStyle = borderStyles[Math.floor(Math.random() * borderStyles.length)]
        const randomColor = '#' + Math.floor(Math.random() * 16777215).toString(16)

        element.style.borderLeft = `${borderWidth}px ${borderStyle} ${randomColor}`
      }
    </script>
  </body>
</html>
```

This example dynamically changes the `borderLeft` of the `div` element to a random width, style, and color each time the button is clicked.

**4. Removing the border-left style:**

```plaintext
<!DOCTYPE html>
<html>
  <head>
    <title>Removing borderLeft</title>
    <style>
      #myDiv {
        width: 200px;
        height: 100px;
        background-color: lightblue;
        border-left: 3px solid green;
      }
    </style>
  </head>
  <body>
    <div id="myDiv">This is a div element.</div>

    <button onclick="removeBorder()">Remove Border Left</button>

    <script>
      function removeBorder() {
        const element = document.getElementById('myDiv')
        element.style.borderLeft = '' // Remove the inline border-left style
      }
    </script>
  </body>
</html>
```

Clicking the button removes the inline `borderLeft` style from the `div` element, causing it to revert to the border defined in the CSS stylesheet (or no border if none is defined).

## Alternatives

While `borderLeft` is convenient for setting all border-left properties at once, you can also use the individual properties for more granular control:

- `borderLeftWidth`: Sets the width of the left border.
- `borderLeftStyle`: Sets the style of the left border.
- `borderLeftColor`: Sets the color of the left border.

For example, instead of:

```javascript
element.style.borderLeft = '2px dashed purple'
```

You could write:

```javascript
element.style.borderLeftWidth = '2px'
element.style.borderLeftStyle = 'dashed'
element.style.borderLeftColor = 'purple'
```

The choice depends on your specific needs and preference. `borderLeft` is more concise when you need to set all three properties, while the individual properties offer greater flexibility when you only need to modify one or two.

## Best Practices

- **Use meaningful values:** Choose descriptive values for width, style, and color to make your code easier to understand.

- **Consider accessibility:** Ensure that your border styles provide sufficient contrast for users with visual impairments.

- **Use CSS classes for complex styles:** For complex or reusable border styles, define them in a CSS class and add/remove the class using JavaScript instead of manipulating the `style` property directly. This promotes separation of concerns and improves maintainability.

- **Avoid inline styles when possible:** While the `borderLeft` property modifies inline styles, it's generally better to define styles in a separate CSS file or `<style>` tag for better organization and maintainability. Use JavaScript to dynamically add or modify CSS classes that define the border styles.

# Mastering `borderRight` in HTML DOM Style: A Comprehensive Guide with Examples

The `borderRight` property in the HTML DOM Style object provides granular control over the right border of any HTML element. Understanding and effectively utilizing this property is crucial for creating visually appealing and dynamic web pages. This article delves deep into `borderRight`, exploring its syntax, possible values, and offering practical JavaScript examples to demonstrate its use in real-world scenarios.

## What is the `borderRight` Property?

The `borderRight` property is a part of the CSS Object Model (CSSOM) accessible through JavaScript. It allows you to programmatically modify the right border of an HTML element. This offers a dynamic way to style your web pages based on user interaction, application state, or other programmatic conditions.

## Syntax

The `borderRight` property is accessed and modified using the following syntax in JavaScript:

```javascript
element.style.borderRight = 'value'
```

Where:

- `element` refers to the HTML element you want to style. This can be retrieved using methods like `document.getElementById()`, `document.querySelector()`, or `document.querySelectorAll()`.
- `style` is the inline style object of the element.
- `borderRight` is the property we're manipulating.
- `value` is a string representing the desired border right style.

## Possible Values

The `borderRight` property accepts a combination of values representing the border's width, style, and color. It can be set in two ways:

**1. Shorthand Notation:**

This is the most common and concise way to define `borderRight`. It allows you to specify all three components (width, style, and color) in a single declaration.

```javascript
element.style.borderRight = '2px solid red'
```

- **`width`:** Specifies the thickness of the border. Possible values include:
  - `thin`
  - `medium`
  - `thick`
  - Length values (e.g., `1px`, `5em`, `0.2rem`)
- **`style`:** Defines the appearance of the border. Common styles include:
  - `none` (Default: No border is displayed.)
  - `dotted` (Border appears as a series of dots.)
  - `dashed` (Border appears as a series of dashes.)
  - `solid` (Border appears as a single, continuous line.)
  - `double` (Border appears as two parallel lines.)
  - `groove` (3D grooved border.)
  - `ridge` (3D ridged border.)
  - `inset` (3D inset border.)
  - `outset` (3D outset border.)
- **`color`:** Specifies the color of the border. Possible values include:
  - Color names (e.g., `red`, `blue`, `green`)
  - Hexadecimal color codes (e.g., `#FF0000`, `#00FF00`, `#0000FF`)
  - RGB values (e.g., `rgb(255, 0, 0)`, `rgb(0, 255, 0)`, `rgb(0, 0, 255)`)
  - RGBA values (e.g., `rgba(255, 0, 0, 0.5)`, `rgba(0, 255, 0, 0.5)`, `rgba(0, 0, 255, 0.5)`)
  - HSL values (e.g., `hsl(0, 100%, 50%)`)
  - HSLA values (e.g., `hsla(0, 100%, 50%, 0.5)`)
  - `transparent`

**2. Longhand Notation:**

You can also control the border right by setting its width, style, and color properties individually:

```javascript
element.style.borderRightWidth = '2px'
element.style.borderRightStyle = 'solid'
element.style.borderRightColor = 'red'
```

While more verbose, this approach allows you to modify only specific aspects of the right border without affecting others.

## Practical Examples

Let's explore some practical JavaScript examples demonstrating how to manipulate the `borderRight` property.

**Example 1: Adding a Solid Right Border on Button Hover**

This example adds a solid right border to a button when the user hovers over it.

```plaintext
<button id="myButton">Hover Me</button>

<script>
  const button = document.getElementById("myButton");

  button.addEventListener("mouseover", function() {
    button.style.borderRight = "3px solid blue";
  });

  button.addEventListener("mouseout", function() {
    button.style.borderRight = "none"; // Remove the border when the mouse leaves
  });
</script>
```

This code first retrieves the button element using its ID. Then, it attaches event listeners for the `mouseover` and `mouseout` events. When the mouse hovers over the button, a `3px` solid blue right border is applied. When the mouse leaves, the border is removed by setting `borderRight` to `none`.

**Example 2: Changing the Right Border Color Dynamically**

This example changes the right border color of a `<div>` element based on user input.

```plaintext
<div id="myDiv" style="border: 1px solid black; width: 200px; height: 100px;">
  This is a div.
</div>
<input type="color" id="colorPicker" value="#000000">

<script>
  const div = document.getElementById("myDiv");
  const colorPicker = document.getElementById("colorPicker");

  colorPicker.addEventListener("input", function() {
    div.style.borderRightColor = colorPicker.value;
  });
</script>
```

This code sets up a `<div>` with a default border and a color picker input. The script listens for changes to the color picker's value and updates the `borderRightColor` of the `<div>` accordingly, allowing the user to dynamically change the border color. It is important to note that the style must initially include `border` settings for the right border to be visible when the color is dynamically changed.

**Example 3: Creating a Dashed Border on Click**

This example adds a dashed right border to a paragraph element when the user clicks on it.

```plaintext
<p id="myParagraph">Click me to add a dashed border!</p>

<script>
  const paragraph = document.getElementById("myParagraph");

  paragraph.addEventListener("click", function() {
    if (paragraph.style.borderRightStyle !== "dashed") {
      paragraph.style.borderRight = "2px dashed green";
    } else {
      paragraph.style.borderRight = "none"; // Remove the border on subsequent clicks
    }
  });
</script>
```

This code toggles the right border style on each click. If the `borderRightStyle` is not already `dashed`, it sets the border to `2px` dashed green. Otherwise, it removes the border. This demonstrates how to check the current style of an element and modify it based on its existing state.

**Example 4: Using `borderRightWidth`, `borderRightStyle` and `borderRightColor`**

This example demonstrates the longhand notation.

```plaintext
<div id="myBox" style="width:100px; height: 100px;"></div>

<script>
    const box = document.getElementById("myBox");

    box.style.borderRightWidth = "5px";
    box.style.borderRightStyle = "dotted";
    box.style.borderRightColor = "purple";
</script>
```

This script creates a `<div>` element and then uses the `borderRightWidth`, `borderRightStyle`, and `borderRightColor` properties individually to style its right border. This provides a more verbose but potentially clearer way to define the border's appearance.

## Best Practices

- **Use Shorthand Notation When Possible:** For simple border definitions, the shorthand notation is more concise and easier to read.
- **Maintain Consistency:** Use consistent border styles and colors throughout your website to create a cohesive visual experience.
- **Consider Accessibility:** Ensure that borders provide sufficient contrast with the background color to be easily visible to users with visual impairments. Avoid relying solely on borders to convey meaning, as they may not be visible to all users. Use additional cues like text labels or icons.
- **Performance Considerations:** While modifying styles dynamically is powerful, excessive DOM manipulation can impact performance. Optimize your code to minimize unnecessary style updates. Consider using CSS classes instead of directly modifying inline styles for more complex styling changes, as changing class names can be more efficient for the browser.
- **Test Across Browsers:** While generally compatible, it's always a good practice to test your website on different browsers to ensure consistent rendering of borders and other CSS styles.

# Common PHP Interview Questions: Ace Your Next PHP Interview

Landing a job as a PHP developer requires more than just knowing the language; it requires demonstrating a solid understanding of its core concepts, frameworks, security considerations, and best practices. This blog post aims to provide a comprehensive guide to common PHP interview questions, equipping you with the knowledge and confidence to excel in your next interview.

## I. Core PHP Concepts

These questions focus on the fundamental building blocks of PHP.

**1. What is PHP and what are its key features?**

This is the most basic question. Your answer should cover:

*   **Definition:** PHP (Hypertext Preprocessor) is a server-side scripting language designed for web development but also used as a general-purpose programming language.
*   **Key Features:**
    *   **Open Source:** Freely available and used by a large community.
    *   **Cross-Platform Compatibility:** Runs on various operating systems (Windows, Linux, macOS).
    *   **Database Connectivity:** Supports various databases like MySQL, PostgreSQL, MongoDB, etc.
    *   **Embedded HTML:** Can be easily embedded within HTML code.
    *   **Object-Oriented Programming (OOP):** Supports OOP principles like inheritance, encapsulation, and polymorphism.
    *   **Large Community Support:** Abundant resources, libraries, and frameworks available.

**2. Explain the difference between `==` and `===` in PHP.**

This tests your understanding of comparison operators.

*   **`==` (Equality Operator):** Compares two variables for equality after *type coercion*. It converts the variables to a common type before comparison.
*   **`===` (Identity Operator):** Compares two variables for equality *without type coercion*. It checks if both the value and the data type are the same.

**Example:**

```php
$a = 10;   // Integer
$b = "10"; // String

if ($a == $b) {
  echo "Equal using ==\n"; // Output: Equal using ==
}

if ($a === $b) {
  echo "Equal using ===\n"; // No output
} else {
  echo "Not equal using ===\n"; // Output: Not equal using ===
}
```

**3. What are the differences between `include` and `require` in PHP?**

This question assesses your knowledge of file inclusion.

*   **`include`:** Includes and evaluates the specified file. If the file is not found, a warning is issued, but the script continues to execute.
*   **`require`:** Includes and evaluates the specified file. If the file is not found, a fatal error is issued, and the script halts.

**`include_once` and `require_once`**:  These are variations that ensure a file is only included or required once, preventing potential errors due to redefined functions or classes.

**When to use which?** Use `require` when the included file is critical for the script's functionality. Use `include` when the included file is optional or when you want to handle the possibility of the file not being present.

**4. Explain the concept of sessions and cookies in PHP.**

This tests your understanding of state management in web applications.

*   **Session:**  A mechanism to store information about a user across multiple pages. Session data is stored on the server, and the user is identified by a unique session ID.
*   **Cookie:** A small text file that a web server stores on a user's computer to remember information about the user.

**Key Differences:**

| Feature       | Session                                   | Cookie                                      |
|---------------|--------------------------------------------|---------------------------------------------|
| Storage       | Server-side                                | Client-side                                 |
| Security      | More secure (data stored on the server)     | Less secure (data stored on the client)    |
| Size          | Can store more data                       | Limited to ~4KB                             |
| Expiration    | Can be set to expire when the browser closes| Can be set to expire after a specific time |

**Example (Session):**

```php
<?php
session_start(); // Start the session

$_SESSION["username"] = "JohnDoe"; // Store username in session

echo "Session variable set.";
?>
```

**Example (Cookie):**

```php
<?php
$cookie_name = "user";
$cookie_value = "John Doe";
setcookie($cookie_name, $cookie_value, time() + (86400 * 30), "/"); // 86400 = 1 day, expire after 30 days

if(!isset($_COOKIE[$cookie_name])) {
  echo "Cookie named '" . $cookie_name . "' is not set!";
} else {
  echo "Cookie '" . $cookie_name . "' is set!<br>";
  echo "Value is: " . $_COOKIE[$cookie_name];
}
?>
```

**5. What are the different types of loops in PHP? Give examples.**

PHP supports several loop types:

*   **`for` loop:**  Executes a block of code a specified number of times.

    ```php
    for ($i = 0; $i < 10; $i++) {
      echo "The number is: " . $i . "<br>";
    }
    ```

*   **`while` loop:**  Executes a block of code as long as a condition is true.

    ```php
    $i = 0;
    while ($i < 10) {
      echo "The number is: " . $i . "<br>";
      $i++;
    }
    ```

*   **`do...while` loop:** Executes a block of code once, and then repeats as long as a condition is true.  The key difference from `while` is that the `do...while` loop is *always* executed at least once.

    ```php
    $i = 0;
    do {
      echo "The number is: " . $i . "<br>";
      $i++;
    } while ($i < 10);
    ```

*   **`foreach` loop:** Iterates over the elements of an array.

    ```php
    $colors = array("red", "green", "blue");
    foreach ($colors as $color) {
      echo "Color: " . $color . "<br>";
    }
    ```

**6. Explain the difference between `GET` and `POST` methods in HTML forms.**

These are fundamental HTTP request methods.

*   **`GET`:** Used to retrieve data from the server.
    *   Data is appended to the URL as query parameters (e.g., `example.com?name=value`).
    *   Visible in the browser's address bar.
    *   Limited data size (typically around 2048 characters).
    *   Not suitable for sensitive data (like passwords).
    *   Can be bookmarked.

*   **`POST`:** Used to submit data to the server to be processed.
    *   Data is sent in the HTTP request body.
    *   Not visible in the browser's address bar.
    *   Larger data size (practically unlimited).
    *   Suitable for sensitive data.
    *   Cannot be bookmarked.

**When to use which?** Use `GET` for retrieving data that doesn't modify the server's state. Use `POST` for submitting data that modifies the server's state, such as creating or updating records.

**7. What are magic constants in PHP? Give some examples.**

Magic constants are predefined constants that change depending on where they are used. They are case-insensitive.  Some common magic constants include:

*   **`__LINE__`:** The current line number of the file.
*   **`__FILE__`:** The full path and filename of the file.
*   **`__DIR__`:** The directory of the file. (PHP 5.3.0 and later)
*   **`__FUNCTION__`:** The name of the function.
*   **`__CLASS__`:** The name of the class.
*   **`__METHOD__`:** The name of the class method.
*   **`__NAMESPACE__`:** The name of the current namespace. (PHP 5.3.0 and later)
*   **`__TRAIT__`**: The name of the trait. (PHP 5.4.0 and later)

**Example:**

```php
<?php
function myFunction() {
  echo "Function name: " . __FUNCTION__ . "<br>";
  echo "Line number: " . __LINE__ . "<br>";
}

myFunction();
?>
```

## II. Object-Oriented Programming (OOP) in PHP

These questions explore your understanding of OOP principles.

**1. Explain the four pillars of OOP.**

*   **Abstraction:** Hiding complex implementation details and showing only essential information to the user.
*   **Encapsulation:** Bundling data (attributes) and methods that operate on that data within a class, protecting data from direct access from outside the class.  This is often achieved using access modifiers (private, protected, public).
*   **Inheritance:** Creating new classes (derived classes) from existing classes (base classes), inheriting their properties and methods.  This promotes code reusability.
*   **Polymorphism:** The ability of an object to take on many forms. This is often achieved through method overriding (in inheritance) and interfaces.

**2. What are access modifiers in PHP and how do they work?**

Access modifiers control the visibility of class members (properties and methods).

*   **`public`:** Accessible from anywhere (inside the class, outside the class, and from derived classes).
*   **`protected`:** Accessible from within the class and from derived classes.
*   **`private`:** Accessible only from within the class itself.

**Example:**

```php
<?php
class MyClass {
  public $public_var = "Public Variable";
  protected $protected_var = "Protected Variable";
  private $private_var = "Private Variable";

  public function showVars() {
    echo $this->public_var . "<br>";
    echo $this->protected_var . "<br>";
    echo $this->private_var . "<br>";
  }
}

$obj = new MyClass();
echo $obj->public_var . "<br>"; // Accessible

// echo $obj->protected_var; // Fatal error: Cannot access protected property
// echo $obj->private_var;   // Fatal error: Cannot access private property

$obj->showVars(); // Accessible within the class
?>
```

**3. What is the difference between an abstract class and an interface in PHP?**

Both abstract classes and interfaces are used to define contracts, but they have key differences.

| Feature       | Abstract Class                                                                   | Interface                                                                          |
|---------------|----------------------------------------------------------------------------------|------------------------------------------------------------------------------------|
| Methods       | Can have both abstract and concrete methods.                                       | Can only have abstract methods (until PHP 8, now interfaces can contain concrete methods). |
| Inheritance   | A class can inherit from only one abstract class.                                | A class can implement multiple interfaces.                                          |
| Properties    | Can have properties.                                                               | Cannot have properties (until PHP 8.1 when constant are permitted).               |
| Access Modifiers | Can use all access modifiers (public, protected, private).                       | All methods are implicitly `public`.                                             |
| Constructor | Can have a constructor.                                                               | Cannot have a constructor.                                                        |

**When to use which?** Use an abstract class when you want to provide some default implementation or when you have properties. Use an interface when you want to define a contract that multiple unrelated classes can adhere to.

**Example (Abstract Class):**

```php
<?php
abstract class Animal {
  abstract public function makeSound();

  public function eat() {
    echo "Eating...<br>";
  }
}

class Dog extends Animal {
  public function makeSound() {
    echo "Woof!<br>";
  }
}

$dog = new Dog();
$dog->makeSound(); // Output: Woof!
$dog->eat();       // Output: Eating...
?>
```

**Example (Interface):**

```php
<?php
interface Eatable {
  public function eat();
}

interface Sleepable {
  public function sleep();
}

class Person implements Eatable, Sleepable {
  public function eat() {
    echo "Eating...<br>";
  }

  public function sleep() {
    echo "Sleeping...<br>";
  }
}

$person = new Person();
$person->eat();   // Output: Eating...
$person->sleep(); // Output: Sleeping...
?>
```

**4. What is method overriding in PHP?**

Method overriding allows a subclass to provide a specific implementation for a method that is already defined in its parent class. The method signature (name and parameters) must be the same.

**Example:**

```php
<?php
class Animal {
  public function makeSound() {
    echo "Generic animal sound...<br>";
  }
}

class Dog extends Animal {
  public function makeSound() { // Method Overriding
    echo "Woof!<br>";
  }
}

$animal = new Animal();
$dog = new Dog();

$animal->makeSound(); // Output: Generic animal sound...
$dog->makeSound();    // Output: Woof!
?>
```

**5. What is the purpose of the `final` keyword in PHP?**

The `final` keyword can be used to prevent a class from being extended (inherited) or a method from being overridden.

*   **`final class`:**  Prevents a class from being subclassed.
*   **`final method`:** Prevents a method from being overridden in a subclass.

**Example:**

```php
<?php
final class FinalClass {
  public function doSomething() {
    echo "Doing something in final class.<br>";
  }
}

// class SubClass extends FinalClass { // Fatal error: Class SubClass may not inherit from final class (FinalClass)
// }

class ParentClass {
  final public function finalMethod() {
    echo "This is a final method.<br>";
  }
}

class ChildClass extends ParentClass {
  // public function finalMethod() { // Fatal error: Cannot override final method ParentClass::finalMethod()
  //   echo "Trying to override final method.<br>";
  // }
}
?>
```

**3. (If you mention Laravel) Explain the concept of Eloquent ORM in Laravel.**

Eloquent is Laravel's ORM (Object-Relational Mapper). It allows you to interact with your database tables as objects.

*   **Model:**  Each database table has a corresponding "Model" that represents it.
*   **CRUD Operations:** Eloquent provides methods for performing CRUD (Create, Read, Update, Delete) operations on database records.
*   **Relationships:** Eloquent makes it easy to define relationships between tables (e.g., one-to-one, one-to-many, many-to-many).
*   **Fluent Interface:** Eloquent uses a fluent interface, making it easy to chain methods together.

**Example:**

```php
<?php
// Assuming you have a model named 'User'

$user->name = 'John Doe';
$user->email = 'john.doe@example.com';

$user->save();

// Delete a user
$user->delete();
?>
```

## IV. Security

Security is a critical aspect of PHP development.

**1. What are some common security vulnerabilities in PHP applications?**

*   **SQL Injection:** Injecting malicious SQL code into database queries.
*   **Cross-Site Scripting (XSS):** Injecting malicious scripts into web pages viewed by other users.
*   **Cross-Site Request Forgery (CSRF):** Tricking a user into performing actions they didn't intend to perform.
*   **File Inclusion Vulnerabilities (Local File Inclusion - LFI, Remote File Inclusion - RFI):**  Exploiting vulnerabilities in file inclusion mechanisms to execute arbitrary code.
*   **Session Hijacking:** Stealing or hijacking a user's session.
*   **Insecure File Uploads:** Allowing users to upload files without proper validation, leading to potential code execution.
*   **Password Storage:** Storing passwords in plain text or using weak hashing algorithms.

**2. How can you prevent SQL injection in PHP?**

*   **Prepared Statements (with Parameter Binding):** The best approach.  Prepared statements separate the SQL query structure from the data, preventing malicious code from being injected.  Use PDO (PHP Data Objects) or MySQLi (MySQL Improved Extension) with prepared statements.
*   **Escaping User Input:**  Use functions like `mysqli_real_escape_string()` to escape special characters in user input before using it in SQL queries.  However, prepared statements are preferred.

**Example (Prepared Statement):**

```php
<?php
$servername = "localhost";
$username = "username";
$password = "password";
$dbname = "myDB";

try {
  $conn = new PDO("mysql:host=$servername;dbname=$dbname", $username, $password);
  // set the PDO error mode to exception
  $conn->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);

  // prepare sql and bind parameters
  $stmt = $conn->prepare("INSERT INTO MyGuests (firstname, lastname, email) VALUES (:firstname, :lastname, :email)");
  $stmt->bindParam(':firstname', $firstname);
  $stmt->bindParam(':lastname', $lastname);
  $stmt->bindParam(':email', $email);

  // insert a row
  $firstname = "John";
  $lastname = "Doe";
  $email = "john@example.com";
  $stmt->execute();

  echo "New records created successfully";
} catch(PDOException $e) {
  echo "Error: " . $e->getMessage();
}
$conn = null;
?>
```

**3. How can you prevent XSS attacks in PHP?**

*   **Output Encoding/Escaping:**  Escape user input when displaying it on the page. Use functions like `htmlspecialchars()` to convert special characters to HTML entities.  Context-aware escaping is important (e.g., escaping for HTML, JavaScript, URLs).
*   **Input Validation:** Validate user input on the server-side to ensure it conforms to expected formats and doesn't contain malicious code.
*   **Content Security Policy (CSP):**  Use CSP headers to control the resources (scripts, stylesheets, images) that the browser is allowed to load.  This can prevent the execution of unauthorized scripts.

**Example (Output Encoding):**

```php
<?php
$username = $_POST["username"];
echo "Welcome, " . htmlspecialchars($username, ENT_QUOTES, 'UTF-8') . "!";
?>
```

**4. How can you prevent CSRF attacks in PHP?**

*   **CSRF Tokens:** Generate a unique, unpredictable token for each user session and include it in forms and AJAX requests.  Verify the token on the server-side before processing the request.  PHP frameworks often provide built-in CSRF protection.
*   **SameSite Cookies:** Use the `SameSite` attribute for cookies to control when cookies are sent in cross-site requests. `SameSite=Strict` prevents cookies from being sent in cross-site requests altogether.  `SameSite=Lax` allows cookies to be sent in cross-site requests only when the user navigates to the site via a top-level GET request.

**5. What is salting and hashing when dealing with passwords?**

*   **Hashing:**  Transforming a password into a fixed-size string of characters using a one-way function.  This makes it difficult to reverse the process and obtain the original password.
*   **Salting:** Adding a random, unique string of characters (the "salt") to each password before hashing it. This prevents attackers from using pre-computed tables of common password hashes (rainbow tables).

**Modern Password Hashing:**  Use `password_hash()` and `password_verify()` functions in PHP. These functions use bcrypt (or Argon2 in newer PHP versions), which are strong hashing algorithms with built-in salting.

**Example (Password Hashing):**

```php
<?php
$password = "mysecretpassword";
$hashedPassword = password_hash($password, PASSWORD_DEFAULT); // Use PASSWORD_DEFAULT for bcrypt

echo "Hashed password: " . $hashedPassword . "<br>";

// Verify the password
$passwordToCheck = "mysecretpassword";
if (password_verify($passwordToCheck, $hashedPassword)) {
  echo "Password is valid!";
} else {
  echo "Invalid password!";
}
?>
```

## V. Best Practices

These questions test your understanding of code quality and maintainability.

**1. What are some coding standards you follow in PHP?**

*   **PSR (PHP Standards Recommendations):**  A set of standards defined by the PHP Framework Interoperability Group (PHP-FIG) that provide guidelines for coding style, autoloading, and other aspects of PHP development.
*   **PSR-1: Basic Coding Standard:**  Defines basic rules for class names, method names, constants, and code formatting.
*   **PSR-2: Coding Style Guide:**  Extends PSR-1 with more detailed rules for code style, including indentation, line length, and comments.
*   **PSR-4: Autoloader:**  Specifies a standard for autoloading classes based on namespaces.
*   **Framework-Specific Standards:**  Many frameworks have their own coding standards that extend or refine the PSR standards.
*   **Consistent Indentation:** Use consistent indentation (typically 4 spaces or 2 spaces) for code blocks.
*   **Meaningful Variable Names:** Use descriptive variable names that clearly indicate the purpose of the variable.
*   **Comments:**  Write clear and concise comments to explain complex logic or non-obvious code.

**2. What is the importance of error handling in PHP?**

*   **Debugging:** Proper error handling helps identify and fix bugs quickly.
*   **User Experience:** Gracefully handle errors to prevent users from seeing cryptic error messages.
*   **Security:**  Prevent sensitive information from being exposed in error messages.
*   **Logging:**  Log errors for monitoring and analysis.

**3. How do you handle errors and exceptions in PHP?**

*   **Error Reporting:**  Configure error reporting levels using `error_reporting()` and `ini_set('display_errors', 1)`.  Be careful about displaying errors in production environments.
*   **Exception Handling:** Use `try...catch` blocks to handle exceptions.
*   **Custom Error Handlers:**  Create custom error handlers using `set_error_handler()` to handle PHP errors in a more controlled way.
*   **Logging:** Log errors and exceptions to a file or database.
*   **Custom Exception Classes:** Create custom exception classes to represent specific error conditions in your application.

**Example (Exception Handling):**

```php
<?php
try {
  // Code that might throw an exception
  $result = 10 / 0; // Division by zero
  echo $result;
} catch (Exception $e) {
  // Handle the exception
  echo "An error occurred: " . $e->getMessage() . "<br>";
  // Log the error
  error_log($e->getMessage());
} finally {
  // Code that will always be executed (optional)
  echo "Finally block executed.<br>";
}
?>
```

**4. What is the purpose of using namespaces in PHP?**

*   **Code Organization:** Namespaces help organize code into logical groups, preventing naming conflicts between classes, functions, and constants.
*   **Code Reusability:** Namespaces make it easier to reuse code across multiple projects.
*   **Autoloading:** Namespaces are often used in conjunction with autoloading to load classes automatically when they are needed.

**Example:**

```php
<?php
namespace MyProject\Utilities;

class StringHelper {
  public static function sanitize($string) {
    return trim(strip_tags($string));
  }
}

namespace AnotherProject;

use MyProject\Utilities\StringHelper;

$input = "<p>  Hello World!  </p>";
$sanitizedString = StringHelper::sanitize($input);
echo $sanitizedString; // Output: Hello World!
?>
```

**5. Explain the concept of Dependency Injection (DI) and Inversion of Control (IoC).**

*   **Dependency Injection (DI):**  A design pattern where dependencies (objects that a class needs to function) are provided to the class from the outside, rather than being created by the class itself.
*   **Inversion of Control (IoC):** A broader principle where the control of object creation and dependency resolution is inverted.  Instead of the application controlling these aspects, a framework or container takes control. Dependency Injection is one way to achieve IoC.

**Benefits of DI/IoC:**

*   **Loose Coupling:** Makes classes less dependent on each other, making the code more flexible and maintainable.
*   **Testability:** Makes it easier to test classes in isolation, as dependencies can be easily mocked or stubbed.
*   **Reusability:** Promotes code reusability, as classes are less tightly coupled to specific implementations.

**Example (Constructor Injection):**

```php
<?php
interface LoggerInterface {
  public function log(string $message): void;
}

class FileLogger implements LoggerInterface {
  public function log(string $message): void {
    // Implementation for logging to a file
    echo "Logging to file: " . $message . "<br>";
  }
}

class UserController {
  private $logger;

  // Constructor Injection
  public function __construct(LoggerInterface $logger) {
    $this->logger = $logger;
  }

  public function createUser(string $username): void {
    // Create a new user
    echo "Creating user: " . $username . "<br>";
    // Log the action
    $this->logger->log("User created: " . $username);
  }
}

// Dependency Injection Container (Simplified Example)
$logger = new FileLogger();
$userController = new UserController($logger); // Inject the FileLogger dependency

$userController->createUser("JohnDoe");
?>
```

## VI. Database Interaction

These questions focus on your ability to interact with databases using PHP.

**1. How do you connect to a database in PHP?**

You can connect to a database using either the `mysqli` extension or PDO (PHP Data Objects). PDO is generally preferred because it provides a consistent interface for connecting to different database systems.

**Example (PDO):**

```php
<?php
$servername = "localhost";
$username = "username";
$password = "password";

try {
  $conn = new PDO("mysql:host=$servername;dbname=myDB", $username, $password);
  // set the PDO error mode to exception
  $conn->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
  echo "Connected successfully";
} catch(PDOException $e) {
  echo "Connection failed: " . $e->getMessage();
}
?>
```

**2. What are the differences between `mysqli` and PDO in PHP?**

| Feature        | mysqli                                           | PDO                                                   |
|----------------|---------------------------------------------------|--------------------------------------------------------|
| Database Support| Specifically designed for MySQL databases.       | Supports a wide range of database systems (MySQL, PostgreSQL, SQLite, Oracle, etc.). |
| Object-Oriented | Both procedural and object-oriented interfaces. | Primarily object-oriented.                            |
| Prepared Statements| Supports prepared statements.                  | Provides a more robust and consistent implementation of prepared statements. |
| Portability   | Less portable to other database systems.          | More portable to other database systems.              |

**3. Explain the different types of JOINs in SQL.**

*   **INNER JOIN:** Returns rows only when there is a match in both tables.
*   **LEFT JOIN (or LEFT OUTER JOIN):** Returns all rows from the left table and the matching rows from the right table. If there is no match in the right table, NULL values are returned for the right table's columns.
*   **RIGHT JOIN (or RIGHT OUTER JOIN):** Returns all rows from the right table and the matching rows from the left table. If there is no match in the left table, NULL values are returned for the left table's columns.
*   **FULL OUTER JOIN:** Returns all rows from both tables. If there is no match, NULL values are returned for the unmatched columns.  (Note:  FULL OUTER JOIN is not directly supported in MySQL; you need to simulate it using UNION).

**4. What is database indexing and why is it important?**

*   **Database Indexing:** Creating an index on one or more columns in a database table to speed up data retrieval.
*   **Importance:**
    *   **Faster Queries:** Indexes allow the database to quickly locate specific rows without scanning the entire table.
    *   **Improved Performance:**  Improves the overall performance of applications that rely on database queries.
    *   **Trade-off:** Indexes consume storage space and can slow down write operations (INSERT, UPDATE, DELETE).

**5. How would you optimize a slow-running SQL query?**

*   **Analyze the Query:** Use `EXPLAIN` to understand how the database is executing the query and identify potential bottlenecks (e.g., full table scans).
*   **Add Indexes:** Add indexes on columns used in `WHERE`, `JOIN`, and `ORDER BY` clauses.
*   **Rewrite the Query:**  Optimize the query logic, such as avoiding subqueries or using more efficient JOINs.
*   **Optimize the Database Schema:**  Consider denormalizing the database schema if necessary.
*   **Use Caching:**  Cache frequently accessed data to reduce the load on the database.
*   **Database Tuning:**  Optimize database server settings (e.g., memory allocation, buffer sizes).

**2. What are Fibers in PHP 8.1 and how can they be used?**

Fibers are a mechanism for achieving cooperative multitasking in PHP. They allow you to pause and resume the execution of code, enabling you to write asynchronous and non-blocking code without relying on external event loops or threads.

*   **Cooperative Multitasking:**  Fibers rely on the code to explicitly yield control (pause execution), allowing other fibers to run.
*   **Asynchronous Programming:**  Fibers can be used to perform I/O operations (e.g., database queries, network requests) without blocking the main thread.
*   **Lightweight Concurrency:** Fibers are much lighter than threads, making them more efficient for handling concurrent tasks.

**Example (Basic Fiber Usage):**

```php
<?php

$fiber = new Fiber(function (): void {
    echo "Fiber started\n";
    Fiber::suspend("Fiber suspended");
    echo "Fiber resumed\n";
});

echo "Before fiber start\n";
$fiber->start();
echo "After fiber start\n";

echo "Before fiber resume\n";
$fiber->resume();
echo "After fiber resume\n";

?>
```

**3. What is the difference between synchronous and asynchronous programming?**

*   **Synchronous Programming:** Code executes sequentially, one instruction at a time. Each operation must complete before the next one can start. Blocking I/O operations can cause the application to freeze or become unresponsive.
*   **Asynchronous Programming:** Allows multiple operations to be performed concurrently. Non-blocking I/O operations are used, and the application can continue processing other tasks while waiting for I/O operations to complete. This can improve performance and responsiveness.

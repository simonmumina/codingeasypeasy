# Expo Calendar: A Comprehensive Guide to Integrating Calendar Functionality in Your React Native Apps

Want to add calendar functionality to your React Native app built with Expo?  This guide walks you through using the `expo-calendar` package, a powerful and convenient way to access and manipulate device calendars on both iOS and Android.  We'll cover everything from installation and permissions to creating, reading, updating, and deleting calendar events.

## What is `expo-calendar`?

The `expo-calendar` package provides a simplified API for interacting with native calendar functionalities within your React Native Expo applications.  It abstracts away the complexities of platform-specific calendar APIs (like EventKit on iOS and CalendarContract on Android), allowing you to write cross-platform code with ease. With `expo-calendar`, you can:

*   **Retrieve lists of calendars:**  Get all the calendars available on the device (e.g., iCloud, Google Calendar, local calendars).
*   **Create new calendars:**  Add new calendars programmatically to the device.
*   **Fetch calendar events:**  Query for events within a specific date range and from specific calendars.
*   **Create new events:**  Add new events to a user's calendar.
*   **Update existing events:**  Modify the details of an existing event.
*   **Delete events:**  Remove events from a user's calendar.
*   **Request calendar permissions:**  Prompt users for access to their calendars.

## Why use `expo-calendar`?

*   **Cross-Platform Compatibility:** Write code once that works on both iOS and Android, saving development time and effort.
*   **Simplified API:**  A clean and straightforward API that is easy to learn and use.
*   **Expo Integration:**  Seamlessly integrates with the Expo ecosystem.
*   **Up-to-Date:**  Maintained and updated to stay compatible with the latest versions of React Native and Expo.

## Getting Started: Installation and Configuration

1.  **Install the Package:**

    First, install the `expo-calendar` package using `expo install`:

    ```bash
    npx expo install expo-calendar
    ```

2.  **Permissions:**

    Before you can access or modify calendars, you need to request permission from the user.  The `expo-calendar` package provides functions to check and request these permissions.  The following code snippet demonstrates how to handle this:

    ```javascript
    import * as Calendar from 'expo-calendar';
    import { useEffect, useState } from 'react';
    import { Button, Text, View } from 'react-native';

    export default function App() {
      const [hasCalendarPermissions, setHasCalendarPermissions] = useState(false);

      useEffect(() => {
        (async () => {
          const { status } = await Calendar.requestCalendarPermissionsAsync();
          setHasCalendarPermissions(status === 'granted');

          if (status === 'granted') {
            console.log('Calendar permissions granted.');
          } else {
            console.log('Calendar permissions denied.');
          }
        })();
      }, []);

      if (!hasCalendarPermissions) {
        return (
          <View style={{ flex: 1, justifyContent: 'center', alignItems: 'center' }}>
            <Text>Calendar permission is required to use this feature.</Text>
            <Button title="Grant Calendar Permission" onPress={async () => {
              const { status } = await Calendar.requestCalendarPermissionsAsync();
              setHasCalendarPermissions(status === 'granted');
            }} />
          </View>
        );
      }

      return (
        <View style={{ flex: 1, justifyContent: 'center', alignItems: 'center' }}>
          <Text>Calendar functionality is ready!</Text>
          {/* Add your calendar interaction code here */}
        </View>
      );
    }
    ```

    **Explanation:**

    *   We import the `Calendar` module from `expo-calendar`.
    *   We use `useEffect` to request calendar permissions when the component mounts.
    *   `Calendar.requestCalendarPermissionsAsync()` prompts the user for permission. The result includes a `status` property indicating whether permission was granted.
    *   We store the permission status in the `hasCalendarPermissions` state variable.
    *   If permission is not granted, we display a message and a button to request permission again.

3.  **Platform-Specific Considerations:**

    *   **iOS:**  Make sure you've added the `CalendarUsageDescription` key to your `app.json` file with a message explaining why your app needs access to the calendar. This is *required* by Apple.

        ```json
        {
          "expo": {
            "ios": {
              "infoPlist": {
                "CalendarUsageDescription": "This app needs access to your calendar to create and manage events."
              }
            }
          }
        }
        ```

    *   **Android:** Permissions are handled more automatically on Android, but you still need to request them properly using the `requestCalendarPermissionsAsync` function.

## Working with Calendars

### Retrieving Calendars

You can retrieve a list of calendars available on the device using `Calendar.getCalendarsAsync()`.

```javascript
import * as Calendar from 'expo-calendar';
import { useEffect, useState } from 'react';
import { Button, Text, View, ScrollView } from 'react-native';

export default function App() {
  const [calendars, setCalendars] = useState([]);
  const [hasCalendarPermissions, setHasCalendarPermissions] = useState(false);

  useEffect(() => {
    (async () => {
      const { status } = await Calendar.requestCalendarPermissionsAsync();
      setHasCalendarPermissions(status === 'granted');

      if (status === 'granted') {
        console.log('Calendar permissions granted.');
        fetchCalendars(); // Call fetchCalendars when permission is granted
      } else {
        console.log('Calendar permissions denied.');
      }
    })();
  }, []);

  const fetchCalendars = async () => {
    if (hasCalendarPermissions) {
      try {
        const fetchedCalendars = await Calendar.getCalendarsAsync(Calendar.DEFAULT);
        setCalendars(fetchedCalendars);
        console.log('Calendars:', fetchedCalendars);
      } catch (error) {
        console.error('Error fetching calendars:', error);
      }
    }
  };


  if (!hasCalendarPermissions) {
      return (
        <View style={{ flex: 1, justifyContent: 'center', alignItems: 'center' }}>
          <Text>Calendar permission is required to use this feature.</Text>
          <Button title="Grant Calendar Permission" onPress={async () => {
            const { status } = await Calendar.requestCalendarPermissionsAsync();
            setHasCalendarPermissions(status === 'granted');
            if (status === 'granted') {
              fetchCalendars();  // Fetch Calendars after permission is granted
            }
          }} />
        </View>
      );
    }

  return (
    <View style={{ flex: 1, padding: 20 }}>
      <Text>Available Calendars:</Text>
      <ScrollView>
        {calendars.map((calendar) => (
          <View key={calendar.id} style={{ marginBottom: 10 }}>
            <Text>Title: {calendar.title}</Text>
            <Text>Source: {calendar.source?.name || 'Unknown'}</Text>
            <Text>Color: {calendar.color}</Text>
            <Text>ID: {calendar.id}</Text>
            {/* Add more calendar details as needed */}
          </View>
        ))}
      </ScrollView>
    </View>
  );
}
```

**Explanation:**

*   We call `Calendar.getCalendarsAsync()` to retrieve the list of calendars. We pass `Calendar.DEFAULT` as the entity types, which returns all available calendars.  You can also use `Calendar.AVAILABILITY` and other constants to filter the types of calendars returned.
*   The result is an array of calendar objects.  Each object contains information about the calendar, such as its `id`, `title`, `color`, and `source`.
*   We iterate over the calendars and display their information in a `ScrollView`.

### Creating a New Calendar

You can create a new calendar using `Calendar.createCalendarAsync()`.  You need to provide several properties for the new calendar, including its name, color, and source.

```javascript
import * as Calendar from 'expo-calendar';
import { useState, useEffect } from 'react';
import { Button, Text, View, Alert } from 'react-native';

export default function App() {
  const [hasCalendarPermissions, setHasCalendarPermissions] = useState(false);
  const [newCalendarId, setNewCalendarId] = useState(null);

  useEffect(() => {
    (async () => {
      const { status } = await Calendar.requestCalendarPermissionsAsync();
      setHasCalendarPermissions(status === 'granted');

      if (status === 'granted') {
        console.log('Calendar permissions granted.');
      } else {
        console.log('Calendar permissions denied.');
      }
    })();
  }, []);

  const createNewCalendar = async () => {
    if (hasCalendarPermissions) {
      try {
        const newCalendar = {
          title: 'My New Calendar',
          color: 'blue',
          entityType: Calendar.EntityTypes.EVENT,
          sourceId: (await getDefaultCalendarSource()).id,  // Important: use the id from getDefaultCalendarSource
          source: await getDefaultCalendarSource(),  // Also provide the whole source object for newer versions
          name: 'my-new-calendar',
          accessLevel: Calendar.CalendarAccessLevel.OWNER,
        };

        const calendarId = await Calendar.createCalendarAsync(newCalendar);
        setNewCalendarId(calendarId);
        Alert.alert('Calendar Created', `Calendar ID: ${calendarId}`);
      } catch (error) {
        console.error('Error creating calendar:', error);
        Alert.alert('Error', 'Failed to create calendar.');
      }
    } else {
      Alert.alert('Permission Denied', 'Calendar permission is required to create a calendar.');
    }
  };

  const getDefaultCalendarSource = async () => {
      const defaultCalendar = await Calendar.getDefaultCalendarAsync();
      return defaultCalendar.source;
  };

  if (!hasCalendarPermissions) {
    return (
      <View style={{ flex: 1, justifyContent: 'center', alignItems: 'center' }}>
        <Text>Calendar permission is required to use this feature.</Text>
        <Button title="Grant Calendar Permission" onPress={async () => {
          const { status } = await Calendar.requestCalendarPermissionsAsync();
          setHasCalendarPermissions(status === 'granted');
        }} />
      </View>
    );
  }

  return (
    <View style={{ flex: 1, justifyContent: 'center', alignItems: 'center' }}>
      <Text>Calendar functionality is ready!</Text>
      <Button title="Create New Calendar" onPress={createNewCalendar} />
      {newCalendarId && <Text>New Calendar ID: {newCalendarId}</Text>}
    </View>
  );
}
```

**Important Considerations for Creating Calendars:**

*   **`sourceId` and `source`:**  Creating a calendar requires a `sourceId` and `source`.  The source represents the account that owns the calendar (e.g., iCloud, Google Calendar).  The best way to get a valid source is to use `getDefaultCalendarSource` as shown in the example.  Providing both the `sourceId` and the `source` object is recommended for compatibility across different Expo versions.
*   **`entityType`:**  Set the `entityType` to `Calendar.EntityTypes.EVENT` for event calendars.
*   **`accessLevel`:** The `accessLevel` determines the level of access your app has to the calendar.  `Calendar.CalendarAccessLevel.OWNER` grants full access.

## Working with Events

### Fetching Events

You can fetch events from a calendar using `Calendar.getEventsAsync()`.  You need to specify the calendars to search, as well as a start and end date for the search range.

```javascript
import * as Calendar from 'expo-calendar';
import { useState, useEffect } from 'react';
import { Button, Text, View, ScrollView } from 'react-native';

export default function App() {
  const [events, setEvents] = useState([]);
  const [hasCalendarPermissions, setHasCalendarPermissions] = useState(false);
  const [defaultCalendarId, setDefaultCalendarId] = useState(null);


  useEffect(() => {
    (async () => {
      const { status } = await Calendar.requestCalendarPermissionsAsync();
      setHasCalendarPermissions(status === 'granted');

      if (status === 'granted') {
        console.log('Calendar permissions granted.');
        getDefaultCalendarId();
      } else {
        console.log('Calendar permissions denied.');
      }
    })();
  }, []);

    const getDefaultCalendarId = async () => {
      const defaultCalendar = await Calendar.getDefaultCalendarAsync();
      setDefaultCalendarId(defaultCalendar.id);
    }


  const fetchEvents = async () => {
    if (hasCalendarPermissions && defaultCalendarId) {
      try {
        const startDate = new Date(); // Today
        const endDate = new Date(Date.now() + 7 * 24 * 60 * 60 * 1000); // One week from now

        const fetchedEvents = await Calendar.getEventsAsync(
          [defaultCalendarId],  // Array of calendar IDs to search
          startDate,
          endDate
        );
        setEvents(fetchedEvents);
        console.log('Events:', fetchedEvents);
      } catch (error) {
        console.error('Error fetching events:', error);
      }
    }
  };


  if (!hasCalendarPermissions) {
    return (
      <View style={{ flex: 1, justifyContent: 'center', alignItems: 'center' }}>
        <Text>Calendar permission is required to use this feature.</Text>
        <Button title="Grant Calendar Permission" onPress={async () => {
          const { status } = await Calendar.requestCalendarPermissionsAsync();
          setHasCalendarPermissions(status === 'granted');
        }} />
      </View>
    );
  }

  return (
    <View style={{ flex: 1, padding: 20 }}>
      <Text>Upcoming Events:</Text>
      <Button title="Fetch Events" onPress={fetchEvents} />
      <ScrollView>
        {events.map((event) => (
          <View key={event.id} style={{ marginBottom: 10 }}>
            <Text>Title: {event.title}</Text>
            <Text>Start Date: {event.startDate}</Text>
            <Text>End Date: {event.endDate}</Text>
            {/* Add more event details as needed */}
          </View>
        ))}
      </ScrollView>
    </View>
  );
}
```

**Explanation:**

*   We define `startDate` and `endDate` to specify the time range for the events we want to fetch.
*   We call `Calendar.getEventsAsync()` with an array containing the calendar IDs to search, the start date, and the end date.
*   The result is an array of event objects, each containing information such as its `id`, `title`, `startDate`, and `endDate`.
*   We iterate over the events and display their information.

### Creating a New Event

You can create a new event using `Calendar.createEventAsync()`.  You need to specify the calendar to add the event to, as well as the event details (e.g., title, start date, end date).

```javascript
import * as Calendar from 'expo-calendar';
import { useState, useEffect } from 'react';
import { Button, Text, View, Alert } from 'react-native';

export default function App() {
  const [hasCalendarPermissions, setHasCalendarPermissions] = useState(false);
  const [newEventId, setNewEventId] = useState(null);

      const defaultCalendar = await Calendar.getDefaultCalendarAsync();
      setDefaultCalendarId(defaultCalendar.id);

  const createNewEvent = async () => {
    if (hasCalendarPermissions && defaultCalendarId) {
      try {
        const startDate = new Date(); // Now
        const endDate = new Date(Date.now() + 60 * 60 * 1000); // One hour from now

        const newEvent = {
          calendarId: defaultCalendarId,
          title: 'My New Event',
          startDate: startDate,
          endDate: endDate,
          timeZone: 'America/New_York', // Specify the timezone

        <Text>Calendar permission is required to use this feature.</Text>

```

**Explanation:**

*   We create a `newEvent` object with the details of the event.

*   `title`, `startDate`, and `endDate` are required properties.
*   `timeZone` is optional, but highly recommended to ensure events are displayed correctly.
*   `allDay` indicates whether the event is an all-day event.
*   We call `Calendar.createEventAsync()` with the `calendarId` and the `newEvent` object. The first argument expects the calendarId.
*   The result is the ID of the newly created event.

## Advanced Usage

### Recurrence Rules

The `expo-calendar` package allows you to create recurring events. You can define a recurrence rule using the `recurrenceRule` property of the event object. While complex recurrence rules may be supported on iOS, Android's support is more limited. Simple recurrence rules like daily, weekly, or monthly are generally well-supported. The following example creates a daily recurring event.

```javascript
const newEvent = {
  calendarId: defaultCalendarId,
  title: 'Daily Recurring Event',
  startDate: new Date(),
  endDate: new Date(Date.now() + 60 * 60 * 1000),
  timeZone: 'America/New_York',
  recurrenceRule: {
    frequency: Calendar.Frequency.DAILY,
    interval: 1, // Repeat every day
    endDate: new Date(Date.now() + 30 * 24 * 60 * 60 * 1000), // Repeat for 30 days
  },
};
```

*   `frequency`: Specifies the frequency of the recurrence (e.g., `Calendar.Frequency.DAILY`, `Calendar.Frequency.WEEKLY`, `Calendar.Frequency.MONTHLY`, `Calendar.Frequency.YEARLY`).
*   `interval`:  Specifies how often the event repeats. For example, an interval of `2` with `frequency: Calendar.Frequency.WEEKLY` means the event repeats every two weeks.
*   `endDate`: Specifies the date after which the event should no longer recur.
*   Other recurrence rule properties include:
    *   `occurrence`: Limits the number of occurrences. Not always supported cross platform.
    *   `daysOfWeek`:  An array of days of the week on which the event should recur (e.g., `[Calendar.DayOfWeek.MONDAY, Calendar.DayOfWeek.WEDNESDAY]`).
    *   `daysOfMonth`: An array of days of the month on which the event should recur (e.g., `[1, 15]`).
    *   `monthsOfYear`: An array of months of the year on which the event should recur (e.g., `[Calendar.Month.JANUARY, Calendar.Month.JULY]`).

### Reminders

You can add reminders to events using the `alarms` property of the event object. Each alarm specifies a time offset before the event when the reminder should trigger.

```javascript
const newEvent = {
  calendarId: defaultCalendarId,
  title: 'Event with Reminder',
  startDate: new Date(),
  endDate: new Date(Date.now() + 60 * 60 * 1000),
  timeZone: 'America/New_York',
  alarms: [
    {
      relativeOffset: -30, // Remind 30 minutes before the event
      method: Calendar.AlarmMethod.DEFAULT
    },
    {
      relativeOffset: -60 * 24, // Remind 1 day before the event
      method: Calendar.AlarmMethod.ALERT
    }
  ],
};
```

*   `relativeOffset`: Specifies the time offset in minutes before the event when the reminder should trigger. A negative value indicates before the event, and a positive value indicates after the event.
*   `method`: Specifies the method used to deliver the reminder. `Calendar.AlarmMethod.DEFAULT` uses the system's default alarm method.  `Calendar.AlarmMethod.ALERT` triggers a visible alert.

### Timezones

Always set timezones properly to prevent events from appearing at the wrong time in different time zones.  Use the `timeZone` property of the event object to specify the timezone.  Ensure you use a valid IANA timezone name, such as `"America/New_York"`, `"Europe/London"`, or `"Asia/Tokyo"`. You can obtain a device's timezone using packages like `expo-localization`.

## Troubleshooting

*   **Permissions Denied:** Ensure you've requested calendar permissions properly and that the user has granted them.  Double-check your `app.json` file to make sure you've added the `CalendarUsageDescription` key on iOS.
*   **Calendar Not Found:**  Make sure the calendar ID you're using is valid.  Use `Calendar.getCalendarsAsync()` to retrieve a list of available calendars and their IDs.
*   **Events Not Showing Up:**  Check the start and end dates you're using when fetching events.  Make sure the events you're looking for fall within that date range.  Also, check the timezone settings of your events and your device.
*   **Source Error During Calendar Creation:** The most common issue is a problem with the `sourceId` or `source`.  Make sure you retrieve the source properly using `getDefaultCalendarSource` as shown in the example.  Pass both the `sourceId` and `source` object for better compatibility.
*   **Android Issues:**  Calendar functionality on Android can be more inconsistent due to the variety of Android devices and calendar apps. Thoroughly test on different Android devices and versions to ensure compatibility.

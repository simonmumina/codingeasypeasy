# Create CSS Pagination: A Complete Guide with Examples

Pagination is an essential component of modern web design, allowing users to navigate through large datasets and content efficiently. While JavaScript libraries and backend frameworks often provide pagination solutions, creating basic pagination with just CSS offers a lightweight and customizable alternative, especially for static sites or situations where minimal scripting is desired. This guide provides a comprehensive walkthrough on building CSS pagination, covering styling, accessibility, and best practices.

## Why CSS Pagination?

Before diving into the implementation, let's consider the benefits of using CSS for pagination:

- **Lightweight:** Eliminates the need for JavaScript, reducing page load times and improving performance.
- **Customizable:** CSS offers extensive styling options to match your website's design and branding.
- **Simple:** Suitable for static websites or when dynamic data updates are handled server-side.
- **Accessible:** When implemented correctly, CSS pagination can be made accessible to users with disabilities.

## Basic CSS Pagination Structure

The fundamental structure of CSS pagination involves using an unordered list (`<ul>`) with list items (`<li>`) representing each page number or navigation link. We'll use anchor tags (`<a>`) within the list items to provide clickable links.

```plaintext
<nav aria-label="Page navigation">
  <ul class="pagination">
    <li class="page-item"><a class="page-link" href="#">Previous</a></li>
    <li class="page-item"><a class="page-link" href="#">1</a></li>
    <li class="page-item"><a class="page-link" href="#">2</a></li>
    <li class="page-item"><a class="page-link" href="#">3</a></li>
    <li class="page-item"><a class="page-link" href="#">Next</a></li>
  </ul>
</nav>
```

**Explanation:**

- `<nav aria-label="Page navigation">`: The `<nav>` element is used for navigation sections, and the `aria-label` attribute provides a descriptive label for screen readers.
- `<ul class="pagination">`: The `<ul>` element represents the list of pagination links. We apply a class `pagination` for styling.
- `<li class="page-item">`: Each `<li>` represents a single pagination item. The `page-item` class is for styling and potential JavaScript interaction later on (if needed).
- `<a class="page-link" href="#">`: The `<a>` element is the actual clickable link to a specific page. `page-link` is our styling class.

## Styling the Pagination with CSS

Now, let's add some CSS to style the pagination. We'll create a clean and modern look.

```plaintext
.pagination {
  display: flex; /* Use flexbox for easy horizontal alignment */
  list-style: none; /* Remove default list styling */
  padding: 0;
  margin: 0;
}

.page-item {
  margin-right: 5px; /* Add spacing between items */
}

.page-link {
  display: block; /* Make the link fill the entire item */
  padding: 8px 12px;
  border: 1px solid #ddd;
  text-decoration: none;
  color: #333;
  background-color: #fff;
  border-radius: 4px;
  transition: background-color 0.3s ease; /* Smooth hover effect */
}

.page-link:hover {
  background-color: #f5f5f5;
}

/* Style the active/current page */
.page-item.active .page-link {
  background-color: #007bff;
  color: #fff;
  border-color: #007bff;
}

/* Style the "disabled" state (e.g., for "Previous" on the first page) */
.page-item.disabled .page-link {
  color: #888;
  background-color: #eee;
  border-color: #ddd;
  pointer-events: none; /* Disable click events */
  cursor: not-allowed; /* Indicate that it's not clickable */
}
```

**CSS Explanation:**

- **`.pagination`**:
  - `display: flex;`: Uses flexbox to arrange the pagination items horizontally.
  - `list-style: none;`: Removes the default bullet points from the unordered list.
  - `padding: 0;` and `margin: 0;`: Resets default padding and margin to ensure consistent layout.
- **`.page-item`**: `margin-right: 5px;`: Adds a small margin to the right of each item for spacing.
- **`.page-link`**:
  - `display: block;`: Makes the anchor tag a block-level element, allowing it to fill the entire list item and be fully clickable.
  - `padding: 8px 12px;`: Adds padding inside the link for better visual appearance.
  - `border: 1px solid #ddd;`: Adds a subtle border.
  - `text-decoration: none;`: Removes the default underline from the link.
  - `color: #333;`: Sets the text color.
  - `background-color: #fff;`: Sets the background color.
  - `border-radius: 4px;`: Adds rounded corners.
  - `transition: background-color 0.3s ease;`: Creates a smooth transition effect on hover.
- **`.page-link:hover`**: Changes the background color on hover.
- **`.page-item.active .page-link`**: Styles the active page link (the currently selected page).
- **`.page-item.disabled .page-link`**: Styles the disabled state, preventing users from clicking "Previous" on the first page or "Next" on the last page.

## Implementing Active and Disabled States

To visually indicate the current page and disable the "Previous" and "Next" links on the first and last pages, we need to add the appropriate classes to the HTML:

```plaintext
<nav aria-label="Page navigation">
  <ul class="pagination">
    <li class="page-item disabled"><a class="page-link" href="#">Previous</a></li>
    <!-- Disabled on first page -->
    <li class="page-item active"><a class="page-link" href="#">1</a></li>
    <!-- Active page -->
    <li class="page-item"><a class="page-link" href="#">2</a></li>
    <li class="page-item"><a class="page-link" href="#">3</a></li>
    <li class="page-item"><a class="page-link" href="#">Next</a></li>
  </ul>
</nav>
```

**Explanation:**

- `class="page-item disabled"`: Adds the `disabled` class to the "Previous" link on the first page. Our CSS rules will then style it appropriately.
- `class="page-item active"`: Adds the `active` class to the list item containing the current page link (in this case, page 1).

**Note:** In a real-world scenario, these classes would be dynamically added and removed using a backend templating language or JavaScript, depending on the current page and total number of pages. This CSS guide focuses on the _styling_ aspect of pagination.

## Accessibility Considerations

Making your pagination accessible is crucial for users with disabilities. Here are some key considerations:

- **`aria-label` on `<nav>`:** As seen in the HTML example, use `aria-label="Page navigation"` on the `<nav>` element to provide a clear description for screen readers.
- **`aria-current="page"` on active link:** Use the `aria-current` attribute on the active link to explicitly indicate that it represents the current page.

  ```plaintext
  <li class="page-item active"><a class="page-link" href="#" aria-current="page">1</a></li>
  ```

- **Semantic HTML:** Use semantic HTML elements like `<nav>`, `<ul>`, `<li>`, and `<a>` correctly to convey the structure and purpose of the pagination.
- **Sufficient Contrast:** Ensure that the text color has sufficient contrast against the background color to be readable for users with visual impairments. Use a contrast checker tool to verify your color choices.
- **Keyboard Navigation:** Make sure users can navigate the pagination using the keyboard (typically using the Tab key). The links should receive focus in a logical order.

## Advanced CSS Pagination Techniques

Let's explore some advanced techniques to enhance your CSS pagination:

### 1. Using `gap` for Spacing (Modern Approach)

Instead of using `margin-right` on each `.page-item`, you can use the `gap` property on the `.pagination` container (requires a modern browser). This simplifies the CSS and avoids potential issues with margins on the last item.

```plaintext
.pagination {
  display: flex;
  list-style: none;
  padding: 0;
  margin: 0;
  gap: 5px; /* Adds spacing between all items */
}

.page-item {
  /* Remove the margin-right from .page-item */
}
```

### 2. Adding Ellipsis for Long Pagination

When dealing with a large number of pages, displaying every page number can clutter the interface. Use ellipses (`...`) to represent skipped page ranges. This usually requires some JavaScript or backend logic to dynamically determine which page numbers to show and where to place the ellipses.

```plaintext
<nav aria-label="Page navigation">
  <ul class="pagination">
    <li class="page-item"><a class="page-link" href="#">Previous</a></li>
    <li class="page-item"><a class="page-link" href="#">1</a></li>
    <li class="page-item disabled"><span class="page-link">...</span></li>
    <!-- Ellipsis -->
    <li class="page-item"><a class="page-link" href="#">5</a></li>
    <li class="page-item"><a class="page-link" href="#">6</a></li>
    <li class="page-item active"><a class="page-link" href="#" aria-current="page">7</a></li>
    <li class="page-item"><a class="page-link" href="#">8</a></li>
    <li class="page-item"><a class="page-link" href="#">9</a></li>
    <li class="page-item disabled"><span class="page-link">...</span></li>
    <!-- Ellipsis -->
    <li class="page-item"><a class="page-link" href="#">10</a></li>
    <li class="page-item"><a class="page-link" href="#">Next</a></li>
  </ul>
</nav>
```

```plaintext
/* Style for the ellipsis */
.page-item.disabled .page-link {
  /* The same styles as the default disabled state */
  color: #888;
  background-color: #eee;
  border-color: #ddd;
  pointer-events: none;
  cursor: not-allowed;
}
```

In this example, the ellipses are represented by `<span>` elements with the `page-link` class and the `disabled` class to prevent interaction.

### 4. Responsive Pagination

Ensure your pagination adapts to different screen sizes. Use media queries to adjust the layout and styling as needed.

```plaintext
@media (max-width: 768px) {
  .pagination {
    flex-wrap: wrap; /* Allow items to wrap to the next line */
  }

  .page-item {
    margin-bottom: 5px;
  }
}
```

This example makes the pagination items wrap to the next line on smaller screens.

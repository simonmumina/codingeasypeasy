# Understanding and Using the `borderRightStyle` HTML DOM Style Object

The `borderRightStyle` property in the HTML DOM Style object allows you to programmatically control the style of the right border of an HTML element using JavaScript. This gives you dynamic control over the visual appearance of your web pages, enabling interactive and responsive designs. This guide delves into the intricacies of this property, providing you with a thorough understanding and practical examples.

## What is the `borderRightStyle` Property?

The `borderRightStyle` property is part of the CSS styling properties that are exposed through the HTML DOM. It determines the visual appearance of the right border of an element. It can accept various values corresponding to different CSS border styles, allowing you to create borders that are solid, dashed, dotted, double, and more.

## Syntax

The syntax for accessing and modifying the `borderRightStyle` property is straightforward:

```plaintext
element.style.borderRightStyle = 'value'
```

Where:

- `element` is a reference to the HTML element you want to style. You typically obtain this reference using methods like `document.getElementById()`, `document.querySelector()`, or `document.getElementsByClassName()`.
- `style` is the inline style object of the element. This is how you access and modify the CSS properties directly through JavaScript.
- `borderRightStyle` is the specific property we're targeting.
- `value` is a string representing the desired border style. Valid values include: `"none"`, `"hidden"`, `"dotted"`, `"dashed"`, `"solid"`, `"double"`, `"groove"`, `"ridge"`, `"inset"`, and `"outset"`.

## Valid Values for `borderRightStyle`

Here's a breakdown of the valid values for the `borderRightStyle` property, along with their visual representations:

- **`none`**: No border is displayed. The computed border width is zero.
- **`hidden`**: Similar to `none`, but may have different conflict-resolution rules for table borders. The computed border width is zero.
- **`dotted`**: Displays a dotted border. The appearance of the dots depends on the user agent.
- **`dashed`**: Displays a dashed border. The appearance of the dashes depends on the user agent.
- **`solid`**: Displays a solid, single-line border.
- **`double`**: Displays a double border. The width of the two lines and the space between them is equal to the specified border width.
- **`groove`**: Displays a 3D grooved border. The effect depends on the border color value.
- **`ridge`**: Displays a 3D ridged border. The effect is the opposite of `groove`, and depends on the border color value.
- **`inset`**: Displays a 3D inset border. The effect depends on the border color value.
- **`outset`**: Displays a 3D outset border. The effect is the opposite of `inset`, and depends on the border color value.
- **`initial`**: Sets the property to its default value (which is `none`).
- **`inherit`**: Inherits this property from its parent element.

## Code Examples

Let's explore several code examples that demonstrate how to use the `borderRightStyle` property to dynamically style HTML elements.

**Example 1: Setting the `borderRightStyle` to "solid"**

```plaintext
<div id="myDiv" style="width: 200px; height: 100px; background-color: lightblue;">
  This is a div element.
</div>

<script>
  const myDiv = document.getElementById("myDiv");
  myDiv.style.borderRightStyle = "solid";
  myDiv.style.borderRightWidth = "5px";
  myDiv.style.borderRightColor = "red";
</script>
```

This code first gets a reference to the `div` element with the ID "myDiv". Then, it sets the `borderRightStyle` to "solid", the `borderRightWidth` to "5px", and `borderRightColor` to "red". This will create a solid red border on the right side of the `div`. Note that you must also set `borderRightWidth` for the border to be visible.

**Example 2: Changing the `borderRightStyle` on Mouseover**

```plaintext
<div id="myDiv" style="width: 200px; height: 100px; background-color: lightblue;">
  Hover over me!
</div>

<script>
  const myDiv = document.getElementById("myDiv");

  myDiv.addEventListener("mouseover", function() {
    myDiv.style.borderRightStyle = "dashed";
    myDiv.style.borderRightWidth = "3px";
    myDiv.style.borderRightColor = "blue";
  });

  myDiv.addEventListener("mouseout", function() {
    myDiv.style.borderRightStyle = "none";
  });
</script>
```

This example demonstrates how to dynamically change the `borderRightStyle` of an element when the mouse hovers over it. When the mouse enters the `div`, the `borderRightStyle` changes to "dashed" with a blue color and 3px width. When the mouse leaves, the `borderRightStyle` is set back to "none", removing the border.

**Example 3: Using a Select Dropdown to Change the `borderRightStyle`**

```plaintext
<select id="borderStyleSelect">
  <option value="none">None</option>
  <option value="solid">Solid</option>
  <option value="dashed">Dashed</option>
  <option value="dotted">Dotted</option>
  <option value="double">Double</option>
</select>

<div id="myDiv" style="width: 200px; height: 100px; background-color: lightblue;">
  Select a border style!
</div>

<script>
  const borderStyleSelect = document.getElementById("borderStyleSelect");
  const myDiv = document.getElementById("myDiv");

  borderStyleSelect.addEventListener("change", function() {
    const selectedStyle = borderStyleSelect.value;
    myDiv.style.borderRightStyle = selectedStyle;
    myDiv.style.borderRightWidth = "2px"; // Ensure width is set
    myDiv.style.borderRightColor = "green"; // Ensure color is set
  });
</script>
```

This example uses a `<select>` dropdown to allow the user to choose the `borderRightStyle`. When the user selects a different option, the `change` event listener is triggered. The script then updates the `borderRightStyle` of the `div` element based on the selected value. We also set the `borderRightWidth` and `borderRightColor` to ensure the border is visible regardless of the selected style.

**Example 4: Inheriting the `borderRightStyle` from the parent element**

```plaintext
<div id="parentDiv" style="borderRightStyle: solid; borderRightWidth: 4px; borderRightColor: purple;">
  <div id="childDiv" style="width: 100px; height: 50px; background-color: yellow; borderRightStyle: inherit;">
    This child div inherits border style.
  </div>
</div>
```

In this example, the `childDiv` is styled to inherit its `borderRightStyle`, `borderRightWidth`, and `borderRightColor` from its parent, the `parentDiv`. The result is the child having the same solid purple border as the parent, on the right side.

## Best Practices

- **Set `borderRightWidth` and `borderRightColor`:** Remember to set the `borderRightWidth` and `borderRightColor` properties in addition to `borderRightStyle` to ensure that the border is visible. If you only set the style, the border might not appear if the width is zero or the color is transparent.
- **Consider User Experience:** When dynamically changing styles, especially on events like `mouseover`, ensure that the changes are subtle and don't cause layout shifts that might be jarring to the user. Use transitions for smoother visual effects.
- **Accessibility:** Be mindful of accessibility when using borders for visual cues. Ensure that sufficient contrast is maintained and that the borders are not the only way to convey information.
- **Use CSS Classes for Complex Styling:** For more complex styling scenarios, it's often better to add or remove CSS classes using JavaScript instead of directly manipulating inline styles. This keeps your JavaScript code cleaner and more maintainable.

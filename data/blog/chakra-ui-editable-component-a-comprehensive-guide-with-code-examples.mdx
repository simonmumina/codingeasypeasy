# Chakra UI Editable Component: A Comprehensive Guide with Code Examples

Chakra UI is a popular component library for React that provides a set of accessible, reusable, and composable UI components.  One of its particularly useful components is the `Editable` component, which allows users to directly edit text fields on the page, enhancing the user experience and providing a more intuitive interface. This blog post dives deep into the `Editable` component, covering its usage, customization, and best practices with plenty of code examples.

## What is the Chakra UI Editable Component?

The `Editable` component in Chakra UI allows users to modify text directly on the screen, similar to how you would edit a text field in a spreadsheet or document editor. This offers a streamlined and intuitive way to update information without navigating to a separate form or modal.

Here's why the `Editable` component is valuable:

*   **Improved User Experience:**  Directly edit text in place, reducing friction and improving the overall user experience.
*   **Reduced Form Submissions:** Avoid unnecessary form submissions for simple text updates.
*   **Real-Time Feedback:**  Users see changes instantly, creating a more responsive and interactive experience.
*   **Accessibility:** Built with accessibility in mind, the `Editable` component supports screen readers and keyboard navigation.

## Getting Started with the Editable Component

Before you start using the `Editable` component, ensure you have Chakra UI installed in your React project. If you haven't already, you can install it using npm or yarn:

```bash
npm install @chakra-ui/react @emotion/react @emotion/styled framer-motion
# or
yarn add @chakra-ui/react @emotion/react @emotion/styled framer-motion
```

Now, let's look at a basic example of how to use the `Editable` component:

```jsx
import { Editable, EditableInput, EditablePreview } from '@chakra-ui/react';
import React, { useState } from 'react';

function BasicEditable() {
  const [value, setValue] = useState('Click to edit');

  return (
    <Editable
      defaultValue={value}
      onSubmit={(newValue) => setValue(newValue)}
    >
      <EditablePreview />
      <EditableInput />
    </Editable>
  );
}

export default BasicEditable;
```

**Explanation:**

*   **Import necessary components:** We import `Editable`, `EditableInput`, and `EditablePreview` from `@chakra-ui/react`.
*   **State Management:** We use `useState` to manage the value of the editable text.  This is crucial for updating the displayed text after the user edits it.
*   **`Editable` Component:** The `Editable` component acts as the container for the input and preview elements.
    *   `defaultValue`:  Sets the initial text displayed.
    *   `onSubmit`: A function that's called when the user finishes editing (by pressing Enter or clicking outside the input).  This function receives the new value as an argument, allowing you to update the state or perform other actions.
*   **`EditablePreview` Component:** Displays the text when the component is not in editing mode.
*   **`EditableInput` Component:**  Displays the input field when the component is in editing mode.

When the user clicks on the `EditablePreview`, the component transitions to editing mode, replacing the preview with the `EditableInput`.  After editing, the `onSubmit` function is called to update the `value` state.

## Customizing the Editable Component

The `Editable` component offers a variety of customization options to suit your specific needs. Let's explore some of the most common customizations:

### 1. Placeholder Text

You can add placeholder text to the `EditableInput` component using the `placeholder` prop:

```jsx
import { Editable, EditableInput, EditablePreview } from '@chakra-ui/react';
import React, { useState } from 'react';

function EditableWithPlaceholder() {
  const [value, setValue] = useState('');

  return (
    <Editable
      defaultValue={value}
      onSubmit={(newValue) => setValue(newValue)}
    >
      <EditablePreview placeholder="Enter your name" />
      <EditableInput placeholder="Enter your name" />
    </Editable>
  );
}

export default EditableWithPlaceholder;
```

### 2. Changing the Input Type

The `EditableInput` component uses a default `text` input type. You can change this to other input types like `email`, `number`, or `password` using the `type` prop:

```jsx
import { Editable, EditableInput, EditablePreview } from '@chakra-ui/react';
import React, { useState } from 'react';

function EditableEmail() {
  const [value, setValue] = useState('');

  return (
    <Editable
      defaultValue={value}
      onSubmit={(newValue) => setValue(newValue)}
    >
      <EditablePreview placeholder="Enter your email" />
      <EditableInput type="email" placeholder="Enter your email" />
    </Editable>
  );
}

export default EditableEmail;
```

### 3. Styling the Editable Component

You can style the `Editable`, `EditablePreview`, and `EditableInput` components using Chakra UI's style props.  Here's how to change the font size, color, and border:

```jsx
import { Editable, EditableInput, EditablePreview, Box } from '@chakra-ui/react';
import React, { useState } from 'react';

function StyledEditable() {
  const [value, setValue] = useState('Click to edit');

  return (
    <Editable
      defaultValue={value}
      onSubmit={(newValue) => setValue(newValue)}
    >
      <EditablePreview
        _hover={{ bg: 'gray.100' }}
        p={2}
        borderRadius="md"
        fontSize="lg"
      />
      <EditableInput
        border="2px"
        borderColor="blue.500"
        _focus={{ borderColor: 'blue.700' }}
        p={2}
        borderRadius="md"
      />
    </Editable>
  );
}

export default StyledEditable;
```

**Explanation:**

*   We use style props like `_hover`, `p`, `borderRadius`, and `fontSize` to style the `EditablePreview`.
*   We use `border`, `borderColor`, `_focus`, `p`, and `borderRadius` to style the `EditableInput`.  The `_focus` prop styles the input when it's focused.

### 4. Controlling Editability with `isDisabled`

You can disable editing using the `isDisabled` prop on the `Editable` component. This is useful when you want to display data that shouldn't be modified.

```jsx
import { Editable, EditableInput, EditablePreview } from '@chakra-ui/react';
import React from 'react';

function DisabledEditable() {
  return (
    <Editable defaultValue="This field is disabled" isDisabled>
      <EditablePreview />
      <EditableInput />
    </Editable>
  );
}

export default DisabledEditable;
```

### 5. Using different input components with `as`

You can use the `as` prop to render different input elements for the `EditableInput`. For example, you can use a `Textarea` instead of a standard `input` for multi-line editing.

```jsx
import { Editable, EditableInput, EditablePreview, Textarea } from '@chakra-ui/react';
import React, { useState } from 'react';

function EditableTextarea() {
  const [value, setValue] = useState('Click to edit');

  return (
    <Editable
      defaultValue={value}
      onSubmit={(newValue) => setValue(newValue)}
    >
      <EditablePreview whiteSpace="pre-wrap"/>
      <EditableInput as={Textarea} />
    </Editable>
  );
}

export default EditableTextarea;
```

**Explanation:**

*   We import the `Textarea` component from Chakra UI.
*   We use the `as` prop on the `EditableInput` component to specify that we want to render a `Textarea` instead of a standard `input`.
*   `whiteSpace="pre-wrap"` on the `EditablePreview`  ensures that whitespace and line breaks are preserved when displaying the preview.

### 6. Cancelling Edits

The `onCancel` prop allows you to handle scenarios where the user cancels the edit (by pressing Escape).

```jsx
import { Editable, EditableInput, EditablePreview } from '@chakra-ui/react';
import React, { useState } from 'react';
import { useToast } from '@chakra-ui/react'

function EditableWithCancel() {
  const [value, setValue] = useState('Click to edit');
  const toast = useToast()

  return (
    <Editable
      defaultValue={value}
      onSubmit={(newValue) => setValue(newValue)}
      onCancel={() => {
        toast({
          title: 'Edit cancelled.',
          status: 'warning',
          duration: 3000,
          isClosable: true,
        })
      }}
    >
      <EditablePreview />
      <EditableInput />
    </Editable>
  );
}

export default EditableWithCancel;
```

**Explanation:**

* We import and use `useToast` hook from Chakra UI to display a toast message when the edit is cancelled. The `onCancel` function is triggered when the user presses the Escape key.

## Best Practices for Using the Editable Component

*   **Provide Clear Visual Cues:**  Make it clear to users that the text is editable.  Consider using a hover effect or an icon to indicate editability.
*   **Handle Errors Gracefully:** Implement error handling in the `onSubmit` function to validate the user's input and provide feedback if necessary.
*   **Consider Accessibility:** Ensure the component is accessible by providing proper labels and keyboard navigation support.  Chakra UI components are generally very accessible, but always double-check!
*   **Use `defaultValue` and Update State:** The `defaultValue` prop provides the initial value. Be sure to update the state in the `onSubmit` function to reflect the changes.
*   **Local vs. Remote Updates:** Decide whether to update the data locally (in the component's state) or remotely (by sending an API request) when the user submits the changes.

## Advanced Use Cases

### 1. Integrating with an API

You can integrate the `Editable` component with an API to persist the changes to a backend database. Here's an example:

```jsx
import { Editable, EditableInput, EditablePreview } from '@chakra-ui/react';
import React, { useState } from 'react';

function EditableWithAPI() {
  const [value, setValue] = useState('Click to edit');

  const handleUpdate = async (newValue) => {
    try {
      // Replace with your actual API endpoint
      const response = await fetch('/api/update-data', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({ value: newValue }),
      });

      if (response.ok) {
        setValue(newValue);
      } else {
        console.error('Failed to update data');
        // Handle error (e.g., display an error message)
      }
    } catch (error) {
      console.error('Error updating data:', error);
      // Handle error
    }
  };

  return (
    <Editable
      defaultValue={value}
      onSubmit={handleUpdate}
    >
      <EditablePreview />
      <EditableInput />
    </Editable>
  );
}

export default EditableWithAPI;
```

**Explanation:**

*   We define an `handleUpdate` function that makes an API call to update the data.
*   We use `fetch` to send a POST request to the API endpoint.
*   We update the local state only if the API request is successful.
*   We include error handling to catch any errors during the API request.

### 2. Creating a Custom Edit Icon

You can create a custom edit icon to provide a visual cue that the text is editable. Here's an example using Chakra UI's `IconButton`:

```jsx
import { Editable, EditableInput, EditablePreview, IconButton, HStack } from '@chakra-ui/react';
import { EditIcon } from '@chakra-ui/icons';
import React, { useState } from 'react';

function EditableWithIcon() {
  const [value, setValue] = useState('Click to edit');
  const [isEditing, setIsEditing] = useState(false);

  const handleEditClick = () => {
    setIsEditing(true);
  };

  const handleSubmit = (newValue) => {
    setValue(newValue);
    setIsEditing(false);
  };

  const handleCancel = () => {
    setIsEditing(false);
  };


  return (
    <HStack>
      <Editable
        defaultValue={value}
        onSubmit={handleSubmit}
        onCancel={handleCancel}
        isDisabled={!isEditing}
        selectOnClick
      >
        <EditablePreview />
        <EditableInput />
      </Editable>
      {!isEditing && (
        <IconButton
          aria-label="Edit"
          icon={<EditIcon />}
          onClick={handleEditClick}
          size="sm"
        />
      )}
    </HStack>
  );
}

export default EditableWithIcon;
```

**Explanation:**

* We use the `useState` hook to track the `isEditing` state, which determines whether the Editable component is in editing mode.
* We render an `IconButton` with an `EditIcon`.  This icon is only visible when not in editing mode.
* The `onClick` handler of the icon sets `isEditing` to `true`, enabling editing.
* The `isDisabled` prop of the `Editable` component is bound to `!isEditing`, disabling the editable field when it is not in edit mode.
* The `selectOnClick` prop on the `Editable` component selects all text inside the input field when the edit mode is triggered.

# HTML DOMTokenList `contains()` Method: Check if a Class Exists with Code Examples

The `DOMTokenList` interface represents a set of space-separated tokens. This is most commonly used via the `classList` property of HTML elements, providing a convenient way to manipulate the classes applied to an element.  One of the crucial methods offered by `DOMTokenList` is `contains()`. This method allows you to easily and efficiently determine whether a specific class name exists within an element's class list.

In this blog post, we'll delve into the `contains()` method, exploring its syntax, usage, and providing numerous code examples to help you understand and implement it effectively in your web development projects.

## Understanding the `DOMTokenList`

Before diving into `contains()`, let's briefly review what `DOMTokenList` is and how it relates to HTML elements.

HTML elements often have a `class` attribute, which is a space-separated list of class names.  In JavaScript, you can access and manipulate this list using the `classList` property of an element. This property returns a `DOMTokenList` object.

For example:

```html
<div id="myDiv" class="red bold large">This is a div.</div>
```

```javascript
const myDiv = document.getElementById('myDiv');
const classList = myDiv.classList;

console.log(classList); // Output: DOMTokenListÂ ['red', 'bold', 'large', value: 'red bold large']
```

## The `contains()` Method

The `contains()` method of the `DOMTokenList` interface checks if a specified token (class name) exists in the list. It returns `true` if the token is present, and `false` otherwise.

**Syntax:**

```javascript
DOMTokenList.contains(token);
```

*   **`token`**: A string representing the class name you want to check for.

**Return Value:**

*   A Boolean value: `true` if the token is found in the list, `false` otherwise.

## Practical Examples of Using `contains()`

Let's illustrate the usage of `contains()` with several practical examples.

**Example 1: Basic Class Existence Check**

```html
<div id="myElement" class="highlight">This is an element.</div>
<button id="checkButton">Check for 'highlight' class</button>
```

```javascript
const myElement = document.getElementById('myElement');
const checkButton = document.getElementById('checkButton');

checkButton.addEventListener('click', () => {
  const hasHighlightClass = myElement.classList.contains('highlight');

  if (hasHighlightClass) {
    alert("The element has the 'highlight' class!");
  } else {
    alert("The element does NOT have the 'highlight' class.");
  }
});
```

In this example, we have a div with the class "highlight".  The JavaScript code retrieves the element and adds a click event listener to a button. When the button is clicked, it uses `contains('highlight')` to check if the "highlight" class exists on the div. An alert message is displayed accordingly.

**Example 2: Dynamically Adding and Checking Classes**

```html
<div id="dynamicElement">This is a dynamic element.</div>
<button id="addButton">Add 'active' class</button>
<button id="checkActiveButton">Check for 'active' class</button>
```

```javascript
const dynamicElement = document.getElementById('dynamicElement');
const addButton = document.getElementById('addButton');
const checkActiveButton = document.getElementById('checkActiveButton');

addButton.addEventListener('click', () => {
  dynamicElement.classList.add('active');
});

checkActiveButton.addEventListener('click', () => {
  const isActive = dynamicElement.classList.contains('active');

  if (isActive) {
    alert("The element IS active!");
  } else {
    alert("The element is NOT active.");
  }
});
```

This example demonstrates how to dynamically add a class using `classList.add()` and then use `contains()` to check if it exists.  Clicking the "Add 'active' class" button adds the "active" class to the div.  The "Check for 'active' class" button then uses `contains('active')` to determine the element's state.

**Example 3: Handling Multiple Classes**

```html
<div id="multiClassElement" class="box shadow rounded">This has multiple classes.</div>
<button id="checkBoxButton">Check for 'box' class</button>
<button id="checkHiddenButton">Check for 'hidden' class</button>
```

```javascript
const multiClassElement = document.getElementById('multiClassElement');
const checkBoxButton = document.getElementById('checkBoxButton');
const checkHiddenButton = document.getElementById('checkHiddenButton');

checkBoxButton.addEventListener('click', () => {
  const hasBoxClass = multiClassElement.classList.contains('box');
  alert(`Element has 'box' class: ${hasBoxClass}`);
});

checkHiddenButton.addEventListener('click', () => {
  const hasHiddenClass = multiClassElement.classList.contains('hidden');
  alert(`Element has 'hidden' class: ${hasHiddenClass}`);
});
```

This example shows how `contains()` works with elements that have multiple classes.  We check for the existence of the "box" class (which is present) and the "hidden" class (which is not present).

**Example 4: Case Sensitivity**

The `contains()` method is case-sensitive.

```html
<div id="caseElement" class="MyClass">Case Sensitivity Test</div>
<button id="checkCaseButton">Check for 'myclass'</button>
```

```javascript
const caseElement = document.getElementById('caseElement');
const checkCaseButton = document.getElementById('checkCaseButton');

checkCaseButton.addEventListener('click', () => {
  const hasLowercaseClass = caseElement.classList.contains('myclass');
  alert(`Element has 'myclass' class: ${hasLowercaseClass}`); // Output: false
});
```

Even though "MyClass" and "myclass" are visually similar, `contains()` will return `false` because the case doesn't match.

**Example 5: Using `contains()` in Conditional Logic**

```html
<div id="statusElement">Inactive</div>
<button id="toggleButton">Toggle Active</button>
```

```javascript
const statusElement = document.getElementById('statusElement');
const toggleButton = document.getElementById('toggleButton');

toggleButton.addEventListener('click', () => {
  if (statusElement.classList.contains('active')) {
    statusElement.classList.remove('active');
    statusElement.textContent = 'Inactive';
  } else {
    statusElement.classList.add('active');
    statusElement.textContent = 'Active';
  }
});
```

This example demonstrates a common use case: using `contains()` in conditional logic to toggle a class on and off. This approach is very common when building interactive UIs, like buttons or toggles that change their appearance and behaviour depending on the presence of a class.

*   **Efficiency:** `DOMTokenList` operations are generally optimized for performance.
*   **Convenience:** Simplifies class management compared to manual string manipulation.

## Alternatives to `DOMTokenList.contains()` (and why to avoid them)

While other methods might seem viable for checking class existence, they often come with drawbacks:

*   **`element.className.indexOf('class') !== -1`:**  This approach uses string manipulation, which can be less efficient and more prone to errors, especially when dealing with complex class names.
*   **Using Regular Expressions:** While regular expressions can be powerful, they are generally overkill for simply checking if a class exists. They can also be less readable and potentially less performant.

The `DOMTokenList.contains()` method is specifically designed for this purpose and offers the best combination of readability, efficiency, and maintainability.
